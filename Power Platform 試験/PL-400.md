### 試験の対象者
この試験の受験者は、Microsoft Power Platform ソリューションの設計、開発、保護、およびトラブルシューティングを行います。アプリケーションの機能強化、カスタム ユーザー エクスペリエンス、システム統合、データ変換、カスタム プロセスの自動化を含む、ソリューションのコンポーネントを実装します。

また機能、境界、および制約の深い理解を含む、Microsoft Power Platform サービスに関する強力な応用知識を持っている必要があります。受験者は、Microsoft Power Platform の アプリケーション ライフサイクル管理 (ALM) 実践の基本を理解している必要があります。

さらに Power Platform サービス、JavaScript、JSON、TypeScript、C#、HTML、.NET/ASP.NET、RESTful Web API、Microsoft Azure、および Microsoft 365 を含む開発経験を持っている必要があります。

### スキルの評価
注：評価される各スキルの下に記載しているリストは、スキルの評価項目です。このリストは、決定的でも網羅的でもありません。

注：試験内容について、ほとんどの場合、プレビュー機能はカバーされず、一部の機能は GA (General Availability) となった際に試験に追加されます。

#### 技術設計の作成 (10-15%)
- 要件を検証し、技術アーキテクチャを設計する
  - ソリューションの技術アーキテクチャを設計、検証する
  - 認証と承認の戦略を設計する
  - すぐに使用できる機能で要件を満たすことができるかどうかを判断する
  - Logic Apps と Power Automate フローをいつ使用するかを決定する
  - サーバーレス コンピューティング、プラグイン、Power Automate をいつ使用するかを決定する
  - 仮想エンティティ データ ソース プロバイダーを構築するタイミングとコネクターを使用するタイミングを決定する
- ソリューション コンポーネントを設計する
  - データ モデルを設計する
  - Power App sの再利用可能なコンポーネントを設計する
  - カスタム コネクターを設計する
  - サーバー サイド コンポーネントを設計する
- Power Platform の拡張ポイントを説明する
  - Bot Framework スキルや Power Automate フローを含む Power Virtual Agent の拡張ポイントを説明する
  - Power BI API、カスタム ビジュアル、Web サイトやその他のアプリケーションへの Power BI アプリの埋め込みを含む Power BI 拡張ポイントを説明する
  - CRUD API やカスタム スタイリングを含む Power Apps ポータルの拡張ポイントを説明する
  - Web リソースとそれらのユーザーを説明する
#### Microsoft Dataverse の構成 (15-20%)
- デプロイメントをサポートするセキュリティを構成する
  - 運用上のセキュリティ問題をトラブルシュートする
  - セキュリティ ロールと列レベルのセキュリティ プロファイルを作成、更新します
  - ビジネス ユニットとチームを構成する
- テーブルと列を実装する
  - テーブルとテーブルオプションを構成する
  - 列を構成する
  - リレーションシップと行動のタイプを構成する
- アプリケーション ライフサイクル管理 (ALM) を実装する
  - ソリューションを作成し、ソリューション コンポーネントを管理する
  - ソリューションをインポート、エクスポートする
  - ソリューションの依存関係を管理する
  - ソリューションやコード資産を含むプロジェクトのソース管理を実装する
  - 環境変数を作成、使用する
  - Package Deployer および関連ツールを用いてパッケージを作成する方法を説明する
  - デプロイメント パッケージを作成する
  - デプロイメントを自動化する
  - Github Actions を実装する
#### Power Apps の作成と構成 (15-20%)
- モデル駆動アプリを作成する
  - モデル駆動アプリを構成する
  - フォームを構成する
  - 列を構成する
  - 視覚化を構成する
  - コマンドやボタンを構成する
- キャンバス アプリを作成する
  - キャンバス アプリを作成、構成する
  - 制御イベントとプロパティを管理する複雑な式を実装する
  - Application Insights を用いてアプリの使用量を分析する
  - 再利用可能なコンポーネント ライブラリをビルドする
- アプリを管理、トラブルシュートする
  - Monitor や他のブラウザベースの診断ツールを用いてアプリの問題をトラブルシュートする
  - App Checker と Solution Checker から結果を解釈する
  - コネクターと API エラーを特定、解決する
  - データのプリロードやクエリの委任を含むアプリのパフォーマンスを最適化する
#### ビジネス プロセス自動化の構成 (5-10%)
- Power Automate を構成する
  - フローを作成、構成する
  - Dataverse コネクター アクションとトリガーを用いてステップを構成する
  - フロー ステップで複雑な式を実装する
  - エラー ハンドリングを実装する
  - コネクターからの JSON 応答を分析したフローをトラブルシュートする
- プロセスを実装する
  - ビジネス プロセス フローを作成、構成する
  - ビジネス ルールを作成、構成する
  - サーバーサイドおよびクライアントサイド コードを用いて、ビジネス プロセス フローを作成、管理、操作する
  - プロセスをトラブルシュートする
#### ユーザー エクスペリエンスを拡張する (10-15%)
- クライアント スクリプトを用いてビジネス ロジックを適用する
  - クライアント API オブジェクト モデルをターゲットとする JavaScript、TypeScript コードを作成する
  - イベント ハンドラーを登録する
  - Dataverse Web API をターゲットとするクライアントサイド スクリプトを作成する
- Power Apps Component Framework (PCF) コンポーネントを作成する
  - PCF コンポーネント ライフサイクルを説明する
  - 新しい PCF コンポーネントを初期化する
  - PCF コンポーネント マニフェストを構成する
  - コンポーネント インターフェースを実装する
  - コンポーネントをパッケージ化、デプロイ、消費する
  - PCF Device、Utility、WebAPI 機能を用いて構成する
  - ローカル テスト ハーネスを用いて PCF コンポーネントをテスト、デバッグする
- コマンド ボタン関数を作成する
  - コマンドを作成する
  - コマンド ボタン ルールとアクションを設計する
  - Ribbon Workbench を用いてコマンド バーを編集する
  - JavaScript ライブラリの依存関係を管理する
#### プラットフォームを拡張する (15-20%)
- プラグインを作成する
  - プラグイン実行パイプラインを説明する
  - プラグインを設計、開発する
  - プラグインをデバッグ、トラブルシュートする
  - プレイメージとポストイメージを用いてビジネス ロジックを実装する
  - 組織サービス API を用いてデータ操作を実行する
  - プラグインのパフォーマンスを最適化する
  - Plug-in Registration Tool を用いてカスタム アセンブリを登録する
  - カスタム アクション メッセージをターゲットとするプラグインを開発する
- カスタム コネクターを作成する
  - API 定義を作成する
  - API セキュリティを構成する
  - 実行時にコネクタの動作を変更するために、ポリシー テンプレートを使用する
  - Azure Functions をカスタム コネクターとして公開する
  - Postman を用いてパブリック API のカスタム コネクターを作成する
- プラットフォーム API を使用する
  - Dataverse Web API もしくは Organization Serviceを用いて、データおよびプロセスと対話する
  - API 制限再試行ポリシーを実装する
  - パフォーマンス、同時実行性、トランザクション、バッチ処理を最適化する
  - 組織の URL およびその他の情報を検出するため、Global Discovery サービスにクエリする、
  - Web API を用いてエンティティ メタデータ操作を実行する
  - OAuth を用いて認証を実行する
- ワークロードを処理する
  - Azure Functions を用いて長時間実行操作を処理する
  - Azure Functions でスケジュールされたイベント駆動型の関数トリガーを構成する
  - Managed ID を用いて Power Platform を認証する
#### 統合を開発する (5-10%)
- イベントを公開、消費する
  - API を用いてイベントを公開する
  - Plug-in Registration Tool を用いてイベントを発行する
  - Webhook、Azure Service Bus、Azure Event Hub を含む サービス エンドポイントを登録する
  - Azure ソリューションの Dataverse リスナーを実装する
  - Power Platform と相互作用する Azure Functions を作成する
- データ同期を実装する
  - エンティティ変更追跡を構成する
  - プラットフォーム API を用いてエンティティ変更レコードを読み取る
  - 代替キーを作成、使用する
