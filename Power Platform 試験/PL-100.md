# PL-100 : Microsoft Power Platform App Maker
Microsoft 認定資格「Microsoft Certified : Power Platform App Maker Associate」の対象試験です。
- Exam PL-100: Microsoft Power Platform App Maker - Learn | Microsoft Docs  
https://docs.microsoft.com/learn/certifications/exams/pl-100

## 試験について (意訳)
**※2023/03/15に英語版が改定 (アップデート) される予定です。**
- Study Guide - xam PL-100: Microsoft Power Platform App Maker
https://query.prod.cms.rt.microsoft.com/cms/api/am/binary/RE4wo07

受験対象者は、自分自身とチームのタスクとプロセスを簡素化および自動化するためのビジネス ソリューションを構想および作成します。また、使用しているデータ、解決する必要がある問題、必要なプロセスとアプリ エクスペリエンスを理解しています。さらに、正式な IT トレーニングを受けていない可能性がありますが、テクノロジーを用いてビジネス上の問題を解決することに慣れています。  
(The candidate envisions and creates business solutions to simplify and automate tasks and processes for themselves and their team. They understand the data that they are working with, the issues that they need to resolve, and the required processes and app experiences. The candidate may not have formal IT training but is comfortable using technology to solve business problems.)

加えて自発的であり、主要な職務の一環として直面する問題を解決することに重点を置いています。Microsoft Power Platform、Microsoft Teams、Microsoft 365 アプリとサービス、サードパーティ ソリューションなどのアプリの機能に接続して使用する場合があります。利用可能なツールの機能と制限を理解し、それらを適用する方法を理解したいという願望も持っています。  
(The candidate is self-directed and focuses on solving problems that they encounter as part of their primary job role. They may connect to and use features of apps including Microsoft Power Platform, Microsoft Teams, Microsoft 365 apps and services, and third-party solutions. The candidate has a desire to understand the capabilities and limitations of available tools and understand how to apply them.)

### 対象となる領域 : 試験で評価されるスキル
試験は定期的にアップデートされ、役割を果たすために必要なスキルが反映されます。試験を受ける時期に応じて、スキル測定目標の 2 つのバージョンが含まれています。  
(Our exams are updated periodically to reflect skills that are required to perform a role. We have included two versions of the Skills Measured objectives depending on when you are taking the exam.)

常に、試験の英語版を最初にアップデートします。一部の試験は他の言語にローカライズされており、英語版がアップデートされてから約 8 週間後にアップデートされます。その他の利用可能な言語は、試験の詳細 Web ページの試験のスケジュールセクションに一覧表示されています。ご希望の言語で試験を利用できない場合は、試験を完了するために 30 分の追加時間をリクエストすることが可能です。  
(We always update the English language version of the exam first. Some exams are localized into other languages, and those are updated approximately eight weeks after the English version is updated. Other available languages are listed in the Schedule Exam section of the Exam Details webpage. If the exam isn't available in your preferred language, you can request an additional 30 minutes to complete the exam.)

注 : 評価される各スキル続いて記載している箇条書きの一覧は、スキルの評価項目 (どのように評価しているかを示す) となります。関連するトピックが試験内でカバーされている場合があります。  
(Note : The bullets that follow each of the skills measured are intended to illustrate how we are assessing that skill. Related topics may be covered in the exam.)

注 : ほとんどの問題は、GA (General Availability) となった機能に関するものです。プレビュー機能が一般的に使用されている場合、それらに関する問題が含まれる場合があります。  
(Note : Most questions cover features that are general availability (GA). The exam may contain questions on Preview features if those features are commonly used.)

### 機能グループ
#### ビジネス ソリューションの設計 (Design business solutions) (20–25%)
- 高度な設計を作成する  
  (Create a high-level design)
  - ビジネス ソリューションをサポートするために必要な既存のデータ ソースを特定する  
    (Identify existing data sources needed to support a business solution)
  - 対象デバイスに期待されるユーザー エクスペリエンスを説明し、Microsoft Power Platform アプリの種類間の違いを説明する  
    (Describe the expected user experience for targeted devices and describe the differences between Microsoft Power Platform app types)
  - プロセスが実行する予想されるロジックとステップを説明する  
    (Describe the expected logic and steps that a process will execute)
  - 新しいデータ ソースの高レベル構造を作成する  
    (Create a high-level structure for a new data source)
- Microsoft Power Platform コンポーネントを特定する  
  (Identify Microsoft Power Platform components)
  - ビジネス ソリューションに必要な Power Apps アプリの種類を決定する  
    (Determine the required Power Apps app type for a business solution)
  - Dataverse for Teams をいつ利用するかを決定する  
    (Determine when to use Dataverse for Teams)
  - 問題のドメインを Microsoft Power Platform ツールにマップする  
    (Map a problem domain to Microsoft Power Platform tools)
  - ビジネス ロジックを実装するためのオプションを特定する  
    (Identify options for implementing business logic)
  - コネクタを説明する  
    (Describe connectors)
  - クラウド フローとデスクトップ フローのユース ケースについて説明する  
    (Describe uses cases for cloud flows and desktop flows)
  - チャットボットのユース ケースについて説明する  
    (Describe use cases for chatbots)
- ビジネス ソリューションのユーザー インターフェイス (UI) を設計する  
  (Design the user interface (UI) for a business solution)
  - コンポーネントの再利用の機会を特定する  
    (Identify opportunities for component reuse)
  - キャンバス アプリの UI 要素を選択する  
    (Select UI elements for canvas apps)
  - 必要なモデル駆動型フォームと Dataverse ビューを特定する  
    (Identify the model-driven forms and Dataverse views that are needed)
- ビジネス ソリューションのレポートとビジュアル化を設計する  
  (Design reporting and visualizations for business solutions)
  - ビュー、Microsoft Power BI ビジュアル化、ダッシュボードなど、ビジネス ソリューションのレポート オプションを選択する  
    (Select reporting options for business solutions including views, Microsoft Power BI visualizations, and dashboards)
  - Power BI ダッシュボードのビジュアル化を定義する  
    (Define visualizations for Power BI dashboards)
  - モデル駆動型ダッシュボードのビジュアル化を定義する  
    (Define visualizations for model-driven dashboards)
  - キャンバス アプリのビジュアル化を定義する
    (Define visualizations for canvas apps)
#### データの分析とビジュアル化 (Analyze and visualize data) (10–15%)
- Power BI ダッシュボードを作成して利用する  
  (Create and consume Power BI dashboards)
  - Power BI サービスを用いて既存のデータセットから簡単なレポートを作成する  
    (Create a simple report from an existing dataset by using Power BI Service)
  - 既存のレポートから Power BI ダッシュボードを作成する  
    (Create Power BI dashboards from existing reports)
  - キャンバス アプリとモデル駆動型アプリに Power BI ダッシュボードとタイルを埋め込む  
    (Embed Power BI dashboards and tiles in canvas apps and model-driven apps)
  - Power BI ダッシュボードを共有する  
    (Share Power BI dashboards)
- AI Builder モデルを説明する  
  (Describe AI Builder models)
  - AI Builder のユースケースを説明する  
    (Describe use cases for AI Builder)
  - 事前構築済みモデルとカスタム モデルの違いについて説明する  
    (Describe differences between prebuilt models and custom models)
  - カスタム モデルのトレーニング プロセスを説明する  
    (Describe the process for training custom models)
  - Microsoft Power Automate または Microsoft Power Apps 内からモデルを利用する  
    (Use a model from within Microsoft Power Automate or Microsoft Power Apps)
#### ビジネス ソリューションの作成 (Create business solutions) (60–65%)
- 開発中に Microsoft Power Platform コンポーネントを管理する  
  (Manage Microsoft Power Platform components during development)
  - Dataverse でパブリッシャーを作成する  
    (Create a publisher in Dataverse)
  - Dataverse ソリューションを作成する 
    (Create a Dataverse solution)
  - Dataverse ソリューションをエクスポート/インポートする  
    (Export or import Dataverse solution)
  - キャンバスアプリ/クラウド フローをエクスポート/インポートする  
    (Export or import a canvas app or a cloud flow)
  - Dataverse ソリューションに既存のアプリとフローを追加する  
    (Add existing apps and flows to a Dataverse solution)
- モデル駆動型アプリを作成する  
  (Create model-driven apps)
  - モデル駆動型アプリを作成する  
    (Create model-driven apps)
  - Dataverse テーブル フォームを作成、構成する  
    (Create and configure Dataverse table forms)
  - Dataverse テーブル ビューを作成、構成する  
    (Create and configure Dataverse table views)
  - モデル駆動型アプリを他のユーザーやグループと共有する  
    (Share model-driven apps with other users and groups)
  - モデル駆動型ダッシュボードを作成、構成する
    (Create and configure model-driven dashboards)
- キャンバス アプリを作成、管理する  
  (Create and manage canvas apps)
  - キャンバス アプリのデータ ソースに接続する  
    (Connect to data sources in canvas apps)
  - App Checker  の結果を解釈する  
    (Interpret App Checker results)
  - キャンバス アプリのバージョンを管理する  
    (Manage versions of canvas apps)
  - キャンバス アプリを公開する  
    (Publish canvas apps)
  - キャンバス アプリを他のユーザーやグループと共有する  
    (Share canvas apps with other users and groups)
  - キャンバス アプリの画面を作成する  
    (Build canvas apps screens)
  - キャンバス アプリのアセットとコンポーネントを画面に追加する  
    (Add canvas app assets and components to screens)
- キャンバス アプリの画面を作成する  
  (Create screens for canvas apps)
  - フォーム、ギャラリー、ボタン、ラベル、入力コントロール、画像、カスタム コントロールをいつ利用するかを決定する  
    (Determine when to use forms, galleries, button, labels, input controls, images, and custom controls)
  - UI 要素を構成する  
    (Configure UI elements)
  - 累乗 Fx 式を実装する
    (Implement Power Fx formulas)
  - コレクションと変数を実装する  
    (Implement collections and variables)
  - キャンバス アプリからクラウド フローを実行する  
    (Run a cloud flow from a canvas app)
- Microsoft データバースを構成する  
  (Configure Microsoft Dataverse)
  - データ モデルに基づいてテーブルとテーブル列を作成する  
    (Create tables and table columns based on a data model)
  - ルックアップを用いてテーブルをリンクする  
    (Link tables by using lookups)
  - ビジネス ルールのユース ケースと機能について説明する  
    (Describe use cases and capabilities of business rules)
  - Dataverse ビジネス ルールを作成する  
    (Create Dataverse business rules)
  - Dataverse が役割ベースのアクセス制御 (RBAC) を利用する方法を説明する  
    (Describe how Dataverse uses role-based access control (RBAC))
  - 既存の Dataverse セキュリティ ロールにテーブル権限を追加する  
    (Add table permissions to existing Dataverse security roles)
- Power Automate フローを作成する  
  (Create Power Automate flows)
  - クラウド フローのトリガーの種類を説明する  
    (Describe types of triggers for cloud flows)
  - クラウド フローを作成する  
    (Create cloud flows)
  - クラウド フローのトリガーを構成する  
    (Configure triggers for cloud flows)
  - フロー アクションを構成する  
    (Configure flow actions)
  - クラウド フローをテストし、フロー チェッカーの結果を解釈する   
    (Test a cloud flow and interpret flow checker results)
  - クラウド フローの条件付きロジックを実装する  
    (Implement conditional logic for a cloud flow)
  - Power Automate と Teams を用いて承認を作成し、承認プロセスを監視する  
    (Create approvals and monitor the approval process by using Power Automate and Teams)
  - クラウド フローを共有する  
    (Share cloud flows)
  - 個人用のデスクトップ フローを作成する  
    (Create a desktop flow for personal use)
- Microsoft Teams で Microsoft Power Virtual Agents チャットボットを作成する  
  (Create Microsoft Power Virtual Agents chatbots in Microsoft Teams)
  - チャットボットのユースケースと機能を特定する  
    (Identify use cases and capabilities for chatbots)
  - トピックとトリガー フレーズを用いたチャットボットを作成する  
    (Create a chatbot that uses topics and trigger phrases)
  - チャットボットをテストする  
    (Test a chatbot)
  - チャットボットを公開する  
    (Publish a chatbot)
