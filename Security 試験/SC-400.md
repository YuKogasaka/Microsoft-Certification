# SC-400 : Microsoft Information Protection Administrator
Microsoft 認定資格「Microsoft Certified : Information Protection Administrator Associate」の対象試験です。
- Exam SC-400: Microsoft Information Protection Administrator - Learn | Microsoft Docs  
https://docs.microsoft.com/learn/certifications/exams/sc-400

## 試験について (意訳)
**※2022/08/01 に改定 (アップデート) されました。**
- Study Guide - Exam SC-400: Microsoft Information Protection Administrator  
https://query.prod.cms.rt.microsoft.com/cms/api/am/binary/RE4Mr80

この試験では、次の技術タスクを達成する能力を測定します : 情報保護を実装します。データ損失防止を実装します。情報ガバナンスを実装します。
(This exam measures your ability to accomplish the following technical tasks: implement information protection; implement data loss prevention; and implement information governance.)

この試験の受験者は、Microsoft のセキュリティとコンプライアンス テクノロジに関する基礎知識を持っている必要があります。情報保護とデータ損失防止の実装に関する経験がテストされます。また、情報ガバナンスを実装する能力についてもテストされます。
(As an exam candidate, you should have a foundational knowledge of Microsoft security and compliance technologies. You will be tested on your experience with implementing information protection and data loss prevention. You will also be tested on your ability to implement information governance.)

Other available languages are listed in the Schedule Exam section of the Exam Details webpage. If the exam isn’t available in your preferred language, you can request an
additional 30 minutes to complete the exam.)

(Note : Most questions cover features that are general availability (GA). The exam may contain questions on Preview features if those features are commonly used.)

### 機能グループ
#### 情報保護の実装 (Implement information protection) (35-40%)
- 機密情報のタイプを作成、管理する  
(Create and manage sensitive information types)
  - 機密情報のタイプを計画する  
(Plan for sensitive information types)
  - 組織の要件に基づく機密情報のタイプを選択する  
(Select a sensitive information type based on an organization's requirements)
  - カスタム機密情報のタイプを作成、管理する  
(Create and manage custom sensitive information types)
  - データが完全に一致するカスタム機密情報のタイプを作成する  
(Create custom sensitive information types with exact data match)
  - ドキュメントのフィンガープリンティングを実装する  
(Implement document fingerprinting)
  - キーワード辞書を作成、利用する  
(Create and use a keyword dictionary)
- トレーニング可能な分類器を作成、管理する  
(Create and manage trainable classifiers)
  - トレーニング可能な分類器を使用するタイミングを特定する  
(Identify when to use trainable classifiers)
  - トレーニング可能な分類器を設計、作成する  
(Design and create a trainable classifier)
  - トレーニング可能な分類器をテストする  
(Test a trainable classifier)
  - 分類器を再トレーニングする  
(Retrain a classifier)
- 機密ラベルを実装、管理する  
(Implement and manage sensitivity labels)
  - 機密ラベルを管理するためのロールと権限を設計、実装する  
(Design and implement roles and permissions for administering sensitivity labels)
  - 機密ラベルを設計、作成する  
(Design and create sensitivity labels)
  - 機密ラベル ポリシーを構成、管理する  
(Configure and manage sensitivity label policies)
  - Microsoft Teams、Microsoft 365 グループ、Microsoft PowerBI、および Microsoft SharePoint サイトに機密ラベルを適用する  
(Apply sensitivity labels to Microsoft Teams, Microsoft 365 groups, Microsoft PowerBI, and Microsoft SharePoint sites)
  - 自動ラベリング ポリシーを構成、公開する  
(Configure and publish auto-labelling policies)
  - コンテンツ エクスプローラーやアクティビティ エクスプローラーなどのラベル分析ツールを用いてデータ分類やラベルの使用量を監視する  
(Monitor data classification and label usage by using label analytics tools such as Content explorer and Activity explorer)
  - AIP 統合ラベリング スキャナーを用いて、オンプレミス データに一括分類を適用する  
(Apply bulk classification to on-premises data by using the AIP unified labelling scanner)
  - 適用された機密ラベルの保護設定とマーキングを管理する  
(Manage protection settings and marking for applied sensitivity labels)
  - 機密ラベルと保護されたコンテンツのレポート、追跡、およびアクセスを管理する  
(Administer reporting, tracking, and access of sensitivity labels and protected content)
  - Azure Purview に既存の機密ラベルを作成、拡張する 
(Create or extend existing sensitivity labels to Azure Purview)
- 電子メール メッセージの暗号化を設計、実装する  
(Design and implement encryption for email messages)
  - Microsoft 365 の利用可能な方法に基づき、電子メール暗号化ソリューションを設計する
(Design an email encryption solution based on methods available in Microsoft 365)
  - Office 365 Message Encryption を実装する  
(Implement Office 365 Message Encryption)
  - Office 365 Advanced Message Encryption を実装する  
(Implement Office 365 Advanced Message Encryption)
#### データ損失防止の実装 (Implement data loss prevention) (30-35%)
- データ損失防止 (DLP) ポリシーを作成、構成する  
(Create and configure data loss prevention (DLP) policies)
  - 組織に DLP ソリューションを推奨する  
  (Recommend a DLP solution for an organization)
  - DLP のアクセス権限を構成する  
(Configure permissions for DLP)
  - DLP ポリシーを作成、テスト、チューニングする  
(Create, test, and tune DLP policies)
  - ポリシーとルールの優先順位に対して DLP を構成する  
(Configure DLP for policy and rule precedence)
  - Microsoft Exchange Online、Microsoft SharePoint Online、Microsoft OneDrive、Microsoft Teams、Microsoft PowerBI、およびオンプレミス リポジトリに対する DLP ポリシーを構成する  
(Configure DLP policies for Microsoft Exchange Online, Microsoft SharePoint Online, Microsoft OneDrive, Microsoft Teams, Microsoft PowerBI, and on-premises repositories)
  - Microsoft Defender for Cloud Apps で使用する DLP ポリシーを構成する
(Configure DLP policies for use in Microsoft Defender for Cloud Apps)
  - DLP ポリシーを使用するように Microsoft Defender for Cloud Apps でファイル ポリシーを構成する
(Configure file policies in Microsoft Defender for Cloud Apps to use DLP policies)
- Microsoft Endpoint DLP を実装、監視する  
(Implement and monitor Microsoft Endpoint DLP)
  - エンドポイントに対する DLP ポリシーを作成、メンテナンスする  
(Create and maintain DLP policies for endpoints)
  - エンドポイント DLP 設定を構成する  
(Configure endpoint DLP settings)
  - デバイス オンボーディングのデプロイ方法を指定する 
(Specify a deployment method for device onboarding)
  - デバイス オンボーディングのエンドポイント要件を特定する  
(Identify endpoint requirements for device onboarding)
  - エンドポイント アクティビティを監視する  
(Monitor endpoint activities)
  - Microsoft Compliance 拡張機能を実装する  
(Implement Microsoft Compliance Extension)
- データ損失防止ポリシーとアクティビティを分析、対応する  
(Analyze and respond to data loss prevention policies and activities)
  - データ損失防止サポートを分析する  
(Analyze data loss prevention reports)
  - Activity エクスプローラーを用いてデータ損失防止アクティビティを分析する  
(Analyze data loss prevention activities by using Activity explorer)
  - Microsoft 365 コンプライアンス センターでデータ損失防止ポリシー違反を修復する  
(Remediate data loss prevention policy violations in the Microsoft 365 compliance center)
  - Microsoft Defender for Cloud Apps でデータ損失防止違反を修復する  
(Remediate data loss prevention violations in Microsoft Defender for Cloud Apps)
#### 情報ガバナンスの実装 (Implement information governance) (25-30%)
- 保持ラベルを用いて、データを保持、削除する  
(Retain and delete data by using retention labels)
  - 保持ラベルを用いた情報の保持と破棄を計画する  
(Plan for information retention and disposition by using retention labels)
  - 保持ラベルを作成する  
(Create retention labels)
  - 適応スコープを構成、管理する
(Configure and manage adaptive scopes)
  - 保持ラベル ポリシーを構成、公開する  
(Configure and publish retention label policies)
  - 自動適用ラベル ポリシーを構成、公開する  
(Configure and publish auto-apply label policies)
- Microsoft 365 のデータ保持を管理する  
(Manage data retention in Microsoft 365)
  - Microsoft SharePoint Online および OneDrive に保持ポリシーを作成、適用する  
(Create and apply retention policies for Microsoft SharePoint Online and OneDrive)
  - Microsoft Teams に保持ポリシーを作成、適用する  
(Create and apply retention policies for Microsoft Teams)
  - 保存ロックを構成する  
(Configure preservation locks)
  - Microsoft 365 で保持されたコンテンツを回復する  
(Recover retained content in Microsoft 365)
  - Microsoft Exchange の保持ポリシーとタグを実装する  
(Implement retention policies and tags in Microsoft Exchange Online)
  - Microsoft Exchange Online のメールボックス ホールドを適用する
(Apply mailbox holds in Microsoft Exchange Online)
  - Microsoft Exchange Online アーカイブ ポリシーを実装する  
(Implement Microsoft Exchange Online archiving policies)
- Microsoft 365 のレコード管理を実装する  
(Implement records management in Microsoft 365)
  - レコード管理を計画する  
(Plan for records management)
  - レコード管理のラベルを構成する  
(Configure labels for records management)
  - ファイル プランを用いて保持要件を管理する  
(Manage retention requirements with a file plan)
  - ファイル プラン記述子を用いて自動保持を構成する  
(Configure automatic retention using file plan descriptors)
  - 保持ラベルとポリシーを用いてレコードを分類する  
(Classify records using retention labels and policies)
  - Microsoft SharePoint Online のインプレース レコード管理を実装する  
(Implement in-place records management in Microsoft SharePoint Online)
  - イベント ベースの保持を管理する  
(Manage event-based retention)
  - レコードの廃棄を管理する  
(Manage disposition of records)
