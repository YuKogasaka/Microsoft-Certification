### 試験の対象者
この試験の受験者は、クラウド サービスとそれらのサービスが Microsoft Azure でどのように提供されるかについての基礎知識を持っている必要があります。この試験は、クラウド ベースのソリューションとサービスを使い始めたばかりの受験者、または Azure を初めて利用する受験者を対象としています。

Azure Fundamentals 試験は、クラウドの概念、Azure サービス、Azur eワークロード、Azure のセキュリティとプライバシー、および Azure の価格設定とサポートに関する知識を証明する機会です。受験者は、ネットワーキング、ストレージ、コンピューティング、アプリケーション サポート、アプリケーション開発の概念など、一般的なテクノロジの概念に精通している必要があります。

Azure Fundamentals は、他の Azure の役割ベース、もしくは特殊な認定の準備に使用できますが、それらのいずれの前提条件でもありません。

### スキルの評価
注：評価される各スキルの下に記載しているリストは、スキルの評価項目です。このリストは、決定的でも網羅的でもありません。

注：試験内容について、ほとんどの場合、プレビュー機能はカバーされず、一部の機能は GA (General Availability) となった際に試験に追加されます。

#### クラウドの概念を説明する (20-25%)
- クラウド サービスを利用する利点と考慮事項を特定する
  - 高可用性、スケーラビリティ、弾力性、機敏性、およびディザスター リカバリーのようなクラウド コンピューティングの利点を特定する
  - 資本的支出 (CapEx) と 運用支出 (OpEx) の違いを特定する
  - 従量課金ベース モデルを説明する
- クラウド サービスのカテゴリ間の違いを説明する
  - 共有責任モデルを説明する
  - Infrastructure-as-a-Service (IaaS) を説明する
  - Platform-as-a-Service (PaaS) を説明する
  - サーバレス コンピューティングを説明する
  - Software-as-a-Service (SaaS) を説明する
  - ユースケースに基づくサービス タイプを特定する
- クラウド コンピューティングのタイプの違いを説明する
  - クラウド コンピューティングを定義する
  - パブリック クラウドを説明する
  - プライベート クラウドを説明する
  - ハイブリッド クラウドを説明する
  - ３ 種類のクラウド コンピューティングを比較、対比する
#### Azure のコア サービスを説明する (15-20%)
- Azure コア アーキテクチャ コンポーネントを説明する
  - リージョンおよびリージョン ペアの利点と利用方法を説明する
  - 可用性ゾーンの利点と利用方法を説明する
  - リソース グループの利点と利用方法を説明する
  - サブスクリプションの利点と利用方法を説明する
  - 管理グループの利点と利用方法を説明する
  - Azure Resource Manager の利点と利用方法を説明する
  - Azure リソースを説明する
- Azure で利用可能なコア リソースを説明する
  - 仮想マシン、Azure App Services、Azure Container Instances (ACI)、Azure Kubernetes Service (AKS)、および Azure Virtual Desktop の利点と利用方法を説明する
  - 仮想ネットワーク、VPN ゲートウェイ、仮想ネットワーク ピアリング、および ExpressRoute の利点と利用方法を説明する
  - コンテナー (Blob) ストレージ, ディスク ストレージ、ファイル ストレージ、およびストレージ層の利点と利用方法を説明する
  - Cosmos DB、Azure SQL Database、Azure Database for MySQL、Azure Database for PostgreSQL、および SQL マネージド インスタンスの利点と利用方法を説明する
  - Azure Marketplace の利点と利用方法を説明する
#### Azure のコア ソリューションと管理ツールを説明する (10-15%)
- Azure で利用可能なコアソリューションを説明する
  - Internet of Things (IoT) Hub、IoT Central、およびAzure Sphere の利点と利用方法を説明する
  - Azure Synapse Analytics、HDInsight、および Azure Databricks の利点と利用方法を説明する
  - Azure Machine Learning、Cognitive Services、および Azure Bot Service の利点と利用方法を説明する
  - Azure Functions および Logic Apps を含むサーバーレス コンピューティング ソリューションの利点と利用方法を説明する
  - Azure DevOps、GitHub、GitHub Actions、および Azure DevTest Labs の利点と利用方法を説明する
- Azure 管理ツールを説明する
  - Azure Portal、Azure PowerShell、Azure CLI、Cloud Shell、および Azure Mobile App の機能と利用方法を説明する
  - Azure Advisor の機能と利用方法を説明する
  - Azure Resource Manager (ARM) テンプレートの機能と利用方法を説明する
  - Azure Monitor の機能と利用方法を説明する
  - Azure Service Health の機能と利用方法を説明する
#### 一般的なセキュリティおよびネットワーク セキュリティ機能を説明する (10-15%)
- Azure セキュリティ機能を説明する
  - ポリシー コンプライアンス、セキュリティ アラート、セキュリティ スコア、および リソースの検疫を含む Azure Security Center の基本機能を説明する
  - Key Vault の機能と利用方法を説明する
  - Azure Sentinel の機能と利用方法を説明する
  - Azure Dedicated Hosts の機能と利用方法を説明する
- Azure ネットワーク セキュリティを説明する
  - 多層防御の概念を説明する
  - ネットワーク セキュリティ グループ (NSG) の機能と利用方法を説明する
  - Azure Firewall の機能と利用方法を説明する
  - Azure DDoS Protection の機能と利用方法を説明する
#### ID、ガバナンス、プライバシー、およびコンプライアンス機能を説明する (20-25%)
- Azure コア ID サービスを説明する
  - 認証と承認の違いについて説明する
  - Azure Active Directory を定義する
  - Azure Active Directory の機能と利用方法を説明する
  - 条件付きアクセス、多要素認証 (MFA)、およびシングル サインオン (SSO) の機能と利用方法を説明する
- Azure ガバナンス機能を説明する
  - ロール ベースのアクセス制御 (RBAC) の機能と利用方法を説明する
  - リソース ロックの機能と利用方法を説明する
  - タグの機能と利用方法を説明する
  - Azure Policy の機能と利用方法を説明する
  - Azure Blueprints の機能と利用方法を説明する
  - Cloud Adoption Framework for Azure を説明する
- プライバシーおよびコンプライアンス リソースを説明する
  - セキュリティ、プライバシー、およびコンプライアンスに関するマイクロソフトのコア原則を説明する
  - Microsoft プライバシー ステートメント、オンライン サービス条件 (OST) および Data Protection Amendmentデータ保護の修正 (DPA) の目的を説明する
  - Trust Center の目的を説明する
  - Azure コンプライアンス ドキュメントの目的を説明する
  - Azure ソブリン リージョン (Azure Government クラウド サービスおよび Azure China クラウド サービス) の目的を説明する
#### Azure のコスト管理および Service Level Agreements を説明する (10-15%)
- コストの計画と管理の方法を説明する
  - コストに影響を与える可能性のある要因 (リソースの種類、サービス、場所、Ingress および Egress トラフィック) を特定する
  - コストを削減できる要因 (占有インスタンス、占有容量、ハイブリッド使用特典、スポット価格) を特定する
  - 料金計算ツール および総保有コスト (TCO) 計算ツールの機能と利用方法を説明する
  - Azure Cost Management の機能と利用方法を説明する
- Azure Service Level Agreements (SLA) およびサービス ライフサイクルを説明する
  - Azure Service Level Agreements (SLA) の目的を説明する
  - SLA に影響を与える可能性のあるアクションを特定する (例: 可用性ゾーン)
  - Azure のサービス ライフサイクルを説明する (パブリック プレビューおよび一般提供 (GA))
