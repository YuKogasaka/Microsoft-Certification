### 試験の対象者
Windows Server Hybrid Administrator Associate 認定の受験者は、Windows Server オンプレミス、ハイブリッド、およびサービスとしてのインフラストラクチャ (IaaS) プラットフォームのワークロードの構成と管理に関する専門知識を持っている必要があります。

この役割の責任として、Windows Server 環境と Azure サービスの統合、およびオンプレミス ネットワークでの Windows Server の管理が含まれます。この役割は、Azure へのワークロードの移行と展開に加えて、Azure 上で Windows Server IaaS ワークロードを管理および維持します。

またこの役割は通常、Azure 管理者、エンタープライズ アーキテクト、Microsoft 365 管理者、およびネットワーク エンジニアと連携します。

この試験の受験者は、オンプレミス、ハイブリッド、およびクラウド テクノロジーを用いて、コア Windows Server ワークロードを管理します。ID、管理、コンピューティング、ネットワーキング、ストレージなどのオンプレミスおよびハイブリッド ソリューションの実装と管理に関する専門知識を持っている必要があります。これらは、Windows Admin Center、PowerShell、Azure Arc、IaaS 仮想マシン管理などの管理ツールとテクノロジーを使用します。

加えて、この試験の受験者は、Windows Server オペレーティング システムの使用経験が豊富である必要があります。

### スキルの評価
注：評価される各スキルの下に記載しているリストは、スキルの評価項目です。このリストは、決定的でも網羅的でもありません。

注：試験内容について、ほとんどの場合、プレビュー機能はカバーされず、一部の機能は GA (General Availability) となった際に試験に追加されます。

#### オンプレミスおよびクラウド環境へ Active Directory ドメイン サービス (AD DS) のデプロイと管理 (30–35%)
- AD DS ドメイン コントローラーをデプロイ、管理する
  - オンプレミス上にドメイン コントローラーをデプロイ、管理する
  - Azure 上にドメイン コントローラーをデプロイ、管理する
  - 読み取り専用ドメイン コントローラー (RODC) をデプロイする
  - Flexible Single Master Operations (FSMO) の役割をトラブルシュートする
- マルチ サイト、マルチ ドメイン、およびマルチ フォレスト環境を構成する
  - フォレストおよびドメインの信頼を構成、管理する
  - AD DS サイトを構成、管理する
  - AD DS レプリケーションを構成、管理する
- AD DS セキュリティ プリンシパルを作成、管理する
  - AD DS ユーザーとグループを作成、管理する
  - マルチ ドメイン、およびマルチ フォレスト シナリオでユーザーとグループを管理する
  - グループ管理サービス アカウント (gMSA) を実装する
  - AD DS、Azure AD DS、Azure AD に Windows Server を参加する
- ハイブリッド ID を実装、管理する
  - Azure AD Connect を実装する
  - Azure AD Connect 同期を管理する
  - Azure AD Connect クラウド同期を実装する
  - Azure AD、AD DS、および Azure AD DS を統合する
  - Azure AD DS を管理する
  - Azure AD Connect Health を管理する
  - オンプレミスおよびハイブリッド環境の認証を管理する
  - AD DS パスワードを構成、管理
- ドメイン ベースのグループ ポリシーを用いて Windows Server を管理する
  - AD DS のグループ ポリシーを実装する
  - AD DS のグループ ポリシーの基本設定を実装する
  - Azure AD DS のグループ ポリシーを実装する
#### ハイブリッド環境の Windows Server とワークロードの管理 (10–15%)
- ハイブリッド環境の Windows Server を管理する
  - Windows Admin Center ゲートウェイ サーバーをデプロイする
  - Windows Admin Center のターゲット マシンを構成する
  - PowerShell Remoting を構成する
  - セカンド ホップ リモーティングの CredSSP や Kerberos 委任を構成する
  - JEA for PowerShell Remoting を構成する
- Azure サービスを用いて Windows Server およびワークロードを管理する
  - Azure Arc を用いて Windows Server を管理する
  - Azure Policy Guest Configuration を割り当てる
  - 非 Azure マシンの Azure 仮想マシン拡張機能を用いて Azure サービスをデプロイする
  - Windows マシンのアップデートを管理する
  - iWindows Servers を Log Analytics に統合する
  - Windows Servers を Azure Security Center に統合する
  - Windows Server を実行する Azure 上の IaaS 仮想マシン (VM) を管理する
  - ハイブリッド ワークロードに Azure Automation を実装する
  - ターゲット VM 上に Runbook を自動化タスクへ作成する
  - IaaS マシンの構成のドリフトを防止するため、DSCを実装する
#### 仮想マシンとコンテナーの管理 (15–20%)
- Hyper-V およびゲスト仮想マシンを管理する
  - VM 拡張セッション モードを有効にする
  - PowerShell Remoting、PowerShell Direct、および HVC.exe を用いて VM を管理する
  - 入れ子になった仮想化を構成する
  - VM メモリを構成する
  - 統合サービスを構成する
  - 個別のデバイスの割り当て (Discrete Device Assignment) を構成する
  - VM リソース グループを構成する
  - VM CPU グループを構成する
  - ハイパーバイザー スケジューリングのタイプを構成する
  - VM チェックポイントを管理する
  - 仮想マシンの高可用性を実装する
  - VHD および VHDX ファイルを管理する
  - Hyper-V ネットワーク アダプターを構成する
  - NIC チーミングを構成する
  - Hyper-V スイッチを構成する
- コンテナーを作成、管理する
  - Windows Server コンテナー イメージを作成する
  - Windows Server cコンテナー イメージを管理する
  - コンテナー ネットワークを構成する
  - コンテナー インスタンスを管理する
- Windows Server を実行する Azure 仮想マシンを管理する
  - データ ディスクを管理する
  - Azure Virtual Machines をリサイズする
  - Azure 仮想マシンの継続的デリバリーを構成する
  - VM への接続を構成する
  - Azure 仮想マシン ネットワーク構成を管理する
#### オンプレミスおよびハイブリッド インフラストラクチャの実装と管理 (15–20%)
- オンプレミスおよびハイブリッド名前解決を実装する
  - DNS を AD DS に統合する
  - ゾーンおよびレコードを作成、管理する
  - DNS フォワーダ/条件付きフォワーダを構成する
  - Windows Server DNS に Azure DNS プライベート ゾーンを統合する
  - DNSSEC を実装する
- オンプレミスおよびハイブリッド シナリオで IP アドレスを管理する
  - IPAM を実装、管理する
  - DHCP サーバーの役割 (オンプレミスのみ) を実装、構成する
  - ハイブリッド環境の IP アドレスの問題を解決する
  - スコープを作成、管理する
  - IP の予約を作成、管理する
  - DHCP 高可用性を実装する
- オンプレミスおよびハイブリッド ネットワーク接続を実装する
  - リモート アクセスの役割を実装、管理する
  - Azure Network Adapter を実装、管理する
  - Azure Extended Network を実装、管理する
  - ネットワーク ポリシー サーバーの役割を実装、管理する
  - Web Application Proxy を実装する
  - Azure Relay を実装する
  - サイト間仮想プライベート ネットワーク (VPN) を実装する
  - Azure Virtual WAN を実装する
  - Azure AD Application Proxy を実装する
#### ストレージおよびファイル サービスの管理 (15–20%)
- Azure File Sync を構成、同期する
  - Azure File Sync サービスを作成する
  - 同期グループを作成する
  - クラウド エンドポイントを作成する
  - サーバーを登録する
  - サーバー エンドポイントを作成する
  - クラウドを使った階層化を構成する
  - File Sync を監視する
  - Azure File Sync に DFS を移行する
- Windows Server ファイル共有を構成、管理する
  - Windows Server ファイル共有アクセスを構成する
  - ファイル スクリーンを構成する
  - ファイル サーバー リソース マネージャー (FSRM) クォータを構成する
  - ブランチ キャッシュを構成する
  - 分散ファイル システム (DFS) を実装、構成する
- Windows Server ストレージを構成する
  - ディスクおよびボリュームを構成する
  - 記憶域スペースを構成を管理する
  - 記憶域レプリカを構成、管理する
  - データ重複除去を構成する
  - SMB ダイレクトを構成する
  - 記憶域サービスの品質 (QoS) を構成する
  - ファイル システムを構成する
