### 試験の対象者
この試験の受験者は、Azure IoT ソリューションのクラウドおよびエッジ コンポーネントを開発する分野の専門知識を持っている必要があります。

Azure IoT Developer の責任には、クラウドサービスやその他のツールを使用した、デバイスのライフサイクルの管理 (セットアップ、構成、およびメンテナンス) が含まれます。 デバイス トポロジ、接続、デバッグ、セキュリティを含む、Azure IoT ソリューションの設計と、IoT 関連のデータ パイプラインを管理、監視、変換するソリューションを実装します。また、Azure IoT Edge コンポーネントをデプロイし、エッジでデバイス ネットワークを構成します。

Azure IoT Developer は、データ エンジニアやその他の利害関係者と協力して、ビジネス統合を成功させます。

この試験の受験者は、データ ストレージ オプション、データ分析、データ処理、Platform-as-a-Service オプションなど、IoT ソリューションを形成する Azure サービスの実装経験が必要です。この役割は、IoT ソリューションのコード部分内の Azure IoT サービス構成設定を認識し、C#、ノード、C、Python を含む少なくとも 1 つの Azure サポート言語で特定の IoT コーディングタスクを実行できる必要があります。

### スキルの評価
注：評価される各スキルの下に記載しているリストは、スキルの評価項目です。このリストは、決定的でも網羅的でもありません。

注：試験内容について、ほとんどの場合、プレビュー機能はカバーされず、一部の機能は GA (General Availability) となった際に試験に追加されます。

#### IoT ソリューション インフラストラクチャの実装 (10-15%)
- Azure IoT Hub を作成、構成する
  - IoT Hub を作成する
  - デバイスを登録する
  - デバイス ツインを構成する
  - configure IoT Hub 価格レベル (SKU) とスケーリングを構成する
- デバイス メッセージングとコミュニケーションをビルドする
  - SDK を用いてメッセージング ソリューションをビルドする (デバイスとサービス)
  - device-to-cloud コミュニケーションを実装する
  - cloud-to-device コミュニケーションを実装する
  - デバイスへのファイル アップロードを構成する
  - メッセージ サイズとスケーリングを最適化する
  - TLS サーバー証明書を用いて IoT Hub を接続する
- 物理 IoT デバイスを構成する
  - デバイスの仕様に基づいた適切なプロトコルを推奨する
  - デバイスのネットワーク、トポロジ、および接続を構成する
#### デバイスのプロビジョニングと管理 (15-20%)
- Device Provisioning Service (DPS) をセットアップする
  - Device Provisioning Service を作成する
  - DPS に新しい登録を作成する
  - IoT Hub を DPS にリンクする
- デバイス ライフサイクルを管理する
  - DPS を用いてデバイスをプロビジョニングする
  - 自動登録のプロビジョニングを解除する
  - デバイスを廃止 (登録解除) する
- IoT Hub を用いて IoT デバイスを管理する
  - IoT Hub デバイス レジストリのデバイスr リストを管理する
  - デバイス ツインのタグとプロパティを修正する
  - IoT Hub Automatic Device Management を用いてデバイス セットを構成する
  - IoT Hub Job と Direct Method を用いてデバイス セットのアクションをトリガーする
  - デバイス管理プリミティブを用いて、デバイス ファームウェアの更新プロセスを実装する
  - モジュール ID を構成する
- Azure IoT Central を用いて IoT デバイスを管理する
  - Azure IoT Central を用いてカスタム デバイス テンプレートを作成する
  - Azure IoT Central のルールとアクションを構成する
  - 操作ビューをカスタマイズする
  - IoT Central からデバイスを追加、管理する
  - デバイス接続とデータ マッピングをトラブルシュートする
  - 業界向けのアプリ テンプレートに基づいてアプリケーションを作成する
  - カスタム アプリケーション テンプレートをエクスポートする
  - カスタム アプリケーション テンプレートに基づいて新しいアプリケーションを作成、管理する
  - デバイス テンプレートのアップグレードとバージョン管理
  - IoT Central ジョブを実行する
#### IoT Edge の実装 (15-20%)
- IoT Edge デバイスをセットアップする
  - IoT Hub のデバイス ID を作成する
  - IoT Edge の IoT デバイスをセットアップする
  - IoT デバイスのコンテナー ランタイムを洗濯、インストールする
  - ホスト システムにモジュール アクセスを実装する
  - IoT Edge ランタイムを更新する
  - DPS を用いて IoT Edge デバイスをプロビジョニングする
- IoT Edge デバイスをデプロイする
  - 展開マニフェストを作成、実装する
  - 単一 IoT Edge デバイス展開を作成する
  - 複数デバイスを対象とする展開を作成する
  - Azure DevOps を用いて継続的展開を作成する
- IoT Edge モジュールを開発する
  - Edge モジュールを作成、カスタマイズする
  - Edge デバイスにカスタム IoT Edge モジュールをデプロイする
  - Azure Marketplace から Edge デバイスに IoT Edge モジュールをデプロイする
  - Azure Container Registry に IoT Edge モジュールを公開する
  - モジュールの構成を定義する
  - IoT Edge モジュール ルーティングを構成する
  - IoT Edge 展開環境を構成する
  - 開発環境で Edge モジュールをデバッグする
- IoT Edge デバイスを構成する
  - 適切なゲートウェイ パターンを選択する
  - IoT Hub と IoT Edge を用いて IoT ゲートウェイをデプロイする
  - IoT Edge 証明書を構成する
  - (ローカル ストレージを含む) オフライン サポートを実装、構成する
#### ビジネス統合の実装 (5-10%)
- アップストリーム、ダウンストリーム システムを統合する
  - 入力および出力接続を設定する
  - ワークフローをトリガーするための IoT Hub ルーティングを設定する
  - データ インターフェース統合をテストする
  - サードパーティのソリューションを統合する
  - ルールやアラートを含むワークフローを構成する
- Develop Azure Digital Twins (ADT) ソリューションを開発する
  - DTDL を用いてデジタル ツインを作成する
  - IoT デバイス データを ADT モデルと関係にマッピングする
  - IoT デバイス メッセージを取り込み、メッセージを ADT に変換する
  - ビジネス ロジックとデータ処理をトリガーするためにルートとエンドポイントを構成する
  - ADT グラフをクエリする
  - グラフ内の ADT エンティティにあるプロファイルを更新する
  - ADT を管理、トラブルシュートする
#### データの処理と管理 (15-20%)
- Azure IoT Hub のメッセージ ルーティングを構成する
  - IoT Hub にメッセージ エンリッチメントを実装する
  - エンドポイントに IoT デバイス メッセージ ルーティングを実装する
  - ルーティング クエリを定義、テストする
  - Event Grid ソースとして IoT Hub を構成する
  - 複数のエンドポイントがある場合に、デフォルトの Event Hub エンドポイントを再構成する
- IoT データのストリーム処理を構成する
  - IoT データのデータとストリーム処理用の ASA を作成する
  - Azure Functions を用いて IoT データを処理、フィルターする
  - ASA のユーザー定義関数と集計を書き込む
  - ASA の Azure Machine Learning 関数を消費する
  - Stream Analytics 出力を構成する
- ASA クエリを作成する
  - IoT Edge で実行する ASA クエリを書き込む
  - クラウドで実行する ASA クエリを書き込む
- Time Series Insights (TSI) を用いてリアルタイム データを処理する
  - TSI 環境を作成する
  - IoT Hub と TSI 環境を接続する
  - Azure portal を用いて TSI 環境の参照データ セットを作成する
  - 時系列モデルの階層、タイプ、およびインスタンス フィールドを実装する
  - Time Series Expression 構文を用いてデータを消費する
#### IoT ソリューションの監視、トラブルシュート、および最適化 (15-20%)
- 正常性監視を構成する
  - IoT Hub のメトリックを構成する
  - Azure IoT Hub の診断ログをセットアップする
  - IoT Hub スケーリング (SKUuni) をプログラムで構成する
  - Azure Monitor を用いてトレースをクエリ、可視化する
  - IoT Hub の Azure Policy 定義を適用する
  - IoT Edge メトリックを収集する
  - Azure IoT Edge から診断を取得する
- デバイス コミュニケーションをトラブルシュートする
  - RDP/SSH を用いてメンテナンス コミュニケーションを確立する
  - Device Stream を用いてメンテナンス コミュニケーションを確立する
  - デバイスのテレメトリが IoT Hub によって受信されていることを確認する
  - デバイス ツインのプロパティ、タグ、および直接メソッドを検証する
  - デバイスの切断と接続をトラブルシュートする
  - IoT Edge デバイスをトラブルシュートする
- パフォーマンスと可用性を確保する
  - ボトルネックを特定し、解決する
  - 各サービスのキャパシティ要件を計算する
  - パフォーマンスおよびストレス テスト用のシミュレートされたデバイス群を作成する
  - メッセージの損失をトラブルシュートする
  - 手動フェールオーバーをテストする
#### セキュリティの実装 (15-20%)
- IoT デバイスとサービスのセキュリティを実装する
  - デバイスとゲートウェイ セキュリティを実装する
  - セキュアな接続を確保する
- Azure Defender for IoT を実装する
  - Azure IoT Hub の Azure Defender for IoT を有効化する
  - セキュリティ モジュールを作成する
  - カスタム アラートを構成する
  - メッセージ送信間隔を構成する
