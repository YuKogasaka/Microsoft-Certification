### 試験の対象者
この試験の受験者は、Microsoft Azure での SAP ソリューションの長期運用に固有の SAP システム ランドスケープと業界標準に関する豊富な経験と知識を持つアーキテクト、もしくはエンジニアである必要があります。

Azure for SAP ワークロードのアーキテクト、もしくはエンジニアの責任には、サービスに関する推奨事項の作成、最適な復元力、パフォーマンス、スケール、プロビジョニング、サイズ、および監視のために必要に応じたリソースの調整が含まれます。

また、クラウド管理者、クラウドDBA、およびクライアントと提携してソリューションを実装します。

加えて、SAP アプリケーション (SAP HANA、S/4HANA、SAP NetWeaver、SAP BW/4HANA、SAP アプリケーションとデータベース用の OS サーバー、Azure ポータル、ARM テンプレート、オペレーティングシステム、仮想化、クラウド インフラストラクチャ、ストレージ構造、高可用性設計、災害復旧設計、データ保護の概念、およびネットワーキング) に関する豊富な経験と知識を持っている必要があります。

この試験では、SAP HANA および Linux 認定に加えて、Azure Administrator Associate、もしくは Azure Solutions Architect Expert 認定資格を取得することを強くお勧めします。

### スキルの評価
注：評価される各スキルの下に記載しているリストは、スキルの評価項目です。このリストは、決定的でも網羅的でもありません。

注：試験内容について、ほとんどの場合、プレビュー機能はカバーされず、一部の機能は GA (General Availability) となった際に試験に追加されます。

#### SAP ワークロードを Azure に移行 (10-15%)
- 既存の SAP ランドスケープのインベントリを作成する
  - ネットワーク インベントリ
  - セキュリティ インベントリ
  - コンピューティング インベントリ
  - オペレーティング システム インベントリ
  - 復元力と可用性のインベントリ
  - SAP Database インベントリ
  - SAP ランドスケープ アーキテクチャ
  - SAP ワークロード パフォーマンス SLA とメトリック
  - 移行に関する考慮事項
- 移行戦略を設計する
  - SAP HANA ハードウェア ディレクトリの認定とサポート
  - Tailored Datacenter Integration (TDI) ソリューションの設計基準
  - Databox
  - HANA System Replication (HSR)
  - Azure Migrate SAP
  - バックアップおよび復元の方法とソリューション
  - 移行のためのインフラストラクチャ最適化
#### SAP ワークロードをサポートする Azure ソリューションの設計 (20-25%)
- ワークロードをサポートする Azure のコア インフラストラクチャ ソリューションを設計する
  - ネットワーク トポロジ要件
  - セキュリティ要件
  - 仮想/ベア メタル
  - コンピューティング
  - オペレーティング システム要件
  - サポートされる SAP 製品/バージョン
  - バックアップと復元の要件ストレージ要件
  - 近接配置グループ
  - インフラストラクチャ要件
- SAP ワークロードをサポートする Azure インフラストラクチャ サービスを設計する
  - バックアップと復元の要件
  - SLA/高可用性
  - データの保護 (LRS/GRS、可用性ゾーン)
  - コンプライアンス
  - モニタリング
  - ライセンス
  - アプリケーション インターフェース
  - 依存関係
- SAP ワークロードをサポートする復元力のある Azure ソリューションを設計する
  - HANA でサポートされる HA/DR モデル (N+N、N+M)
  - アプリケーション サーバー (NetWeaver および XS Advanced)
  - SAP Central サービス
  - 可用性セット
  - 可用性ゾーン
  - ペア リージョンでの災害復旧 (DR)
  - Database HA
#### SAP ワークロードにおける Azure へのビルドとデプロイ (35-40%)
- 仮想マシン (VM) のデプロイメントを自動化する
  - Azure Resource Manager (ARM) テンプレート
  - VM の構成を自動化する
  - スクリプト開発、スクリプト変更、デプロイメントの依存関係など、自動化ツールを使用したスクリプティング
- 仮想ネットワークを実装、管理する
  - IDS/IPS for Azure
  - ルーティングの基本
  - サブネット戦略
  - SAP ランドスケープの分離とセグメンテーション
- Azureでのアクセスと認証を管理する
  - カスタム ドメイン
  - Azure AD Identity Protection
  - Azure AD Join
  - 条件付きアクセスポリシー
  - ロールベースのアクセス制御 (RBAC)
  - サービス プリンシパル
  - Just in Time アクセス
- ID を実装、管理する
  - Azure AD Connect
  - AD フェデレーション とシングル サインオン
  - LDAP/Kerberos/SSH
  - Linux VM の Active Directory ドメイン メンバーシップ メカニズム
  - SAP HANA Cloud Platform Identity Authentication (Azure AD との統合)
- SAP ワークロードを監視する
  - Azure Extension for SAP
  - Azure Monitor
  - Log Analytics ワークスペースとメトリック
- SAP 製品の HA/DR インフラストラクチャをビルド、デプロイする
  - ASCS/SCS deployments on Linux および Windows 上に ASCS/SCS をデプロイする (SOFS with S2D、Azure NetApp Files、共有ストレージをエミュレートするサードパーティ製品)
  - HA/DR scenarios for SAP HANA の HA/DR シナリオ
  - AnyDB の HA/DR シナリオ
  - SAP Business One、SAP Business Object BI のような非 NetWeaver HA 製品
  - ロード バランサーを利用する場所と接続性のトラブルシューティング
#### SAP ワークロードにおける Azure インフラストラクチャの検証 (10-15%)
- インフラストラクチャ検証チェックを実行する
  - Apache JMeter、Spirent Avalanche、Microfocus LoadRunner
  - SAP ワークロードの実装をテストする
  - ネットワーク パフォーマンスとスループットを検証する
  - ストレージを検証する
  - SAP HANA Hardware と Cloud Measurement Tools (HCMT) (HANA)
  - 柔軟な I/O テスター (FIO) と DD
- 運用準備チェックを実行します
  - バックアップと復元
  - 高可用性のチェック
  - フェールオーバーのテスト
  - DR のテスト
  - 印刷のテスト
#### Azure SAP アーキテクチャの運用化 (10-15%)
- パフォーマンスを最適化する
  - ABAP メーターを用いた Azure 上の SAP ワークロードのパフォーマンスと正常性をテストする
  - ストレージ構造
  - Azure 上の SAP ワークロードをサポートする前提条件
  - 計画停止の定期メンテナンス
  - 計画されていない停止の回復プラン
  - SAPアプリケーションとインフラストラクチャのハウスキーピング (例 : OS ボリュームのスナップショット)
  - Express Route の帯域幅調整
  - IPtables ExpressRoute Fast Path および ExpressRoute GlobalReach for HANA Large Instances (HLI)
- SAP ワークロードを Azure へ移行する
  - 移行戦略
  - Azure Migrate
  - プライベート/パブリック IP アドレス、ネットワーク ルート、ネットワーク インターフェース。サブネット、仮想ネットワーク
  - ストレージ構成
  - ソース環境とターゲット環境の準備
  - データのバックアップと復元
  - SQL Server Migration Assistant (SSMA) (DB2、Oracle、SAP ASE から Azure VM 上の SQL Server への移行)
