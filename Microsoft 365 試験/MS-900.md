# MS-900 : Microsoft 365 Fundamentals
Microsoft 認定資格「Microsoft 365 Certified : Fundamentals」の対象試験です。
- Exam MS-900 : Microsoft 365 Fundamentals - Learn | Microsoft Docs  
https://docs.microsoft.com/learn/certifications/exams/ms-900

## 試験について (意訳)
**※2022/10/19 に改定 (アップデート) 予定です。**
- Study Guide - Exam MS-900: Microsoft 365 Fundamentals  
https://query.prod.cms.rt.microsoft.com/cms/api/am/binary/RE3VwVw

この試験では、組織の IT に関する一般的な課題に対処する Microsoft 365 ソリューションを推奨する能力を評価します。Microsoft 365 ソリューションが生産性を向上させ、コラボレーションを促進、コミュニケーションを最適化する方法を理解する必要があります。

試験の受験者は、Microsoft 365 ソリューションを市場の競合他社と差別化するために必要な知識を持っている必要があります。また、Microsoft 365 のライセンス、展開と移行の支援、およびクラウドへの投資を最大限に活用しようとしている組織向けのサポート オプションについても理解しておく必要があります。

### 対象となる領域 : 試験で評価されるスキル
一部の試験は他の言語にローカライズされており、英語版が改定 (アップデート) されてから約 8 週間後に改定 (アップデート) されます。その他の利用可能な言語は、**試験の詳細** Web ページの**試験のスケジュール**セクションに一覧表示されています。ご希望の言語で試験を受けられない場合は、試験を完了するために 30 分の追加時間のリクエストが可能です。

注 : 評価される各スキル続いて記載している稼業書きの一覧は、スキルの評価項目 (どのように評価しているかを示す) となります。関連するトピックが試験内でカバーされている亜愛があります。

注 : ほとんどの問題は、GA (General Availability) となった機能に関するものです。プレビュー機能が一般的に私用されている場合、それらに関する問題が含まれる場合があります。

### 機能グループ
#### クラウドの概念を説明する (Describe cloud concepts) (10–15%)
- 利用可能なさまざまな種類のクラウド サービスについて説明する  
(Describe the different types of cloud services available)
  - Microsoft SaaS、IaaS、および PaaS が提供する機能を説明する  
(Describe Microsoft SaaS, IaaS, and PaaS offerings)
  - 組織や個人向けのクラウドベースの生産性ソリューションを説明する  
(Cloud-based productivity solutions for organizations and individuals)
  - Office 365、Microsoft 365、および Windows 365 の違いを説明する  
(Describe differences between Office 365, Microsoft 365, and Windows 365)
- クラウド、ハイブリッド、もしくはオンプレミス サービスを使用する利点と考慮事項を説明する  
(Describe the benefits of and considerations for using cloud, hybrid, or onpremises services)
  - パブリック、プライベート、およびハイブリッド シナリオを説明する  
(Describe public, private, and hybrid scenarios)
  - クラウド ベースのサービスとオンプレミス サービスの利点を比較する  
(Compare advantages of cloud-based services versus on-premises services)
  - 組織をクラウドに移行させる要因を特定する  
(Identify factors that drive organizations to move to the cloud)
  - ハイブリッド ワーキング戦略、柔軟なワーキング戦略、および最前線のワーカーについて説明する  
(Describe hybrid working strategies, flexible working strategies, and frontline workers)
#### コア Microsoft 365 サービスと概念を説明する (Describe core Microsoft 365 services and concepts) (50–55%)
- Microsoft 365 の生産性ソリューションについて説明する  
(Describe productivity solutions in Microsoft 365)
  - Microsoft Outlook と Microsoft Exchange、Microsoft 365 アプリ、OneDrive を含む Microsoft 365 のコア生産性とチームワーク機能について説明する  
(Describe the core productivity and teamwork capabilities in Microsoft 365 including Microsoft Outlook and Microsoft Exchange, Microsoft 365 apps, and OneDrive)
  - Word、Excel、PowerPoint、Outlook、および OneNote を含む Microsoft 365 アプリを説明する  
(Describe Microsoft 365 Apps including Word, Excel, PowerPoint, Outlook, and OneNote)
  - Project、Planner、Bookings、ToDo (タスク) を含む Microsoft 365 の作業管理機能について説明する  
(Describe work management capabilities in Microsoft 365 including Project, Planner, Bookings, and ToDo (Tasks))
  - Microsoft Exchange を用いてビジネス クラスの電子メールおよびカレンダー ソリューションについて説明する  
(Describe business class email and calendaring solutions with Microsoft Exchange)
- Microsoft 365 のコラボレーション ソリューションについて説明する  
(Describe collaboration solutions in Microsoft 365)
  - Microsoft Teams、Teams Phone、Yammer、SharePoint、Stream を含む Microsoft 365 のコラボレーション機能について説明する  
(Describe the collaboration capabilities in Microsoft 365 including Microsoft Teams, Teams Phone, Yammer, SharePoint, and Stream)
  - Microsoft Teams のコラボレーション ワークロードとそれらが提供する価値について説明する  
(Describe the collaboration workloads of Microsoft Teams and the value that they provide)
  - Microsoft Viva のコア従業員エクスペリエンス機能について説明する  
(Describe the core employee experience capabilities in Microsoft Viva)
  - コラボレーション アプリを用いて Microsoft Teams を拡張する方法を説明する  
(Describe the ways that you can extend Microsoft Teams by using collaborative apps)
  - コラボレーションを促進する Microsoft SharePoint と Microsoft OneDrive の機能について説明する  
(Describe the features of Microsoft SharePoint and Microsoft OneDrive that promote collaboration)
  - Yammer がコミュニティの接続と成長をどのように支援するかを説明する  
(Describe how Yammer helps communities connect and grow)
- Microsoft 365 のエンドポイントの最新化、管理の概念、および展開オプションについて説明する  
(Describe endpoint modernization, management concepts, and deployment options in Microsoft 365)
  - Microsoft Endpoint Manager (MEM)、Windows 365 Cloud PC、Azure Virtual Desktop を含む Microsoft 365 のエンドポイント管理機能について説明する  
(Describe the endpoint management capabilities of Microsoft 365 including Microsoft Endpoint Manager (MEM), Windows 365 Cloud PC, and Azure Virtual Desktop)
  - どちらかを実装することが理にかなっている場合に Windows 365 と Azure Virtual Desktop の機能を比較する  
(Compare the capabilities of Windows 365 and Azure Virtual Desktop and when it makes sense to implement either)
  - 展開リングを含む Windows-as-a-Service (WaaS) の展開モデルとリリース モデルについて説明する  
(Describe the deployment and release models for Windows-as-a-Service (WaaS) including deployment rings)
  - Microsoft 365 アプリの展開とサービスの方法を特定する  
(Identify deployment and servicing methods for Microsoft 365 apps)
- Microsoft 365 の分析機能を説明する  
(Describe analytics capabilities in Microsoft 365)
  - Viva Insights の機能を説明する  
(Describe the capabilities of Viva Insights)
  - Microsoft 365 管理センターや Microsoft 365 ユーザー ポータルの機能を説明する  
(Describe the capabilities of the Microsoft 365 Admin center and Microsoft 365 user portal)
  - Microsoft 365 管理センターやほかの管理センターで利用可能なサポートを説明する  
(Describe the reports available in the Microsoft 365 Admin center and other admin centers)
#### Microsoft 365 のセキュリティ、コンプライアンス、プライバシー、信頼を説明する (Describe security, compliance, privacy, and trust in Microsoft 365) (15–20%)
- Microsoft 365 のゼロトラスト セキュリティの原則を説明する  
(Explain zero-trust security principles for Microsoft 365)
  - 組織のクラウドおよびオンプレミス インフラストラクチャ内で保護する必要がある主要なコンポーネントを特定する  
(Identify key components that need to be protected within an organization’s cloud and onpremises infrastructure)
  - ゼロトラスト モデルを説明する  
(Describe the Zero-Trust model)
- Microsoft 365 の ID およびアクセス管理ソリューションについて説明する  
(Describe identity and access management solutions in Microsoft 365)
  - Azure Active Directory や Microsoft Entra の一部、および Azure Identity を含む Microsoft 365 の ID およびアクセス管理機能について説明する  
(Describe the identity and access management capabilities in Microsoft 365 including Azure Active Directory, part of Microsoft Entra, and Azure Identity)
  - クラウド ID、オンプレミス ID、およびハイブリッド ID の概念を説明する  
(Describe cloud identity, on-premises identity, and hybrid identity concepts)
  - 多要素認証 (MFA) を実装する目的と価値を説明する  
(Describe the purpose and value of implementing multi-factor authentication (MFA))
  - 条件付きアクセスの目的と価値を説明する  
(Describe the purpose and value of conditional access)
- Microsoft 365 の脅威保護ソリューションについて説明する  
(Describe threat protection solutions in Microsoft 365)
  - Microsoft 365 Defender、Defender for Endpoint、Defender for Office 365、Defender for Identity、および Microsoft 365 Defender Portal について説明する  
(Describe Microsoft 365 Defender, Defender for Endpoint, Defender for Office 365, Defender for Identity, and the Microsoft 365 Defender Portal)
  - Microsoft Secure Score の利点と機能について説明する  
(Describe Microsoft Secure Score benefits and capabilities)
  - エンドポイント、アプリケーション、および ID に対する最も一般的なタイプの脅威について説明する  
(Describe the most common types of threats against endpoints, applications, and identities)
  - Microsoft が最も一般的な脅威にどのように対処するかを説明する  
(Describe how Microsoft addresses the most common threats)
  - Microsoft 365 と Azure Sentinel を用いた、人工知能による脅威の軽減を自動化する方法を説明する  
(Describe how threat mitigation can be automated with artificial intelligence by using Microsoft 365 and Azure Sentinel)
- Microsoft 365 の信頼、プライバシー、コンプライアンス ソリューションについて説明する  
(Describe trust, privacy, and compliance solutions in Microsoft 365)
  - Microsoft 365 のコンプライアンス ソリューションを説明する  
(Describe compliance solutions in Microsoft 365)
  - Service Trust ポータルと Microsoft Purview Compliance Manager の違いを特定する  
(Identify differences between the Service Trust portal and Microsoft Purview Compliance Manager)
  - 規制コンプライアンスを確保するために Microsoft がデータ常駐をサポートする方法を説明する  
(Describe how Microsoft supports data residency to ensure regulatory compliance)
  - データ損失防止 (DLP) 分類ラベルを含む情報保護およびガバナンスのオプションについて説明する  
(Describe information protection and governance options including data loss prevention (DLP) classification labels)
  - Microsoft Purview Compliance Manager、Compliance Scores、Microsoft Priva、およびそれらが組織に提供できるメリットについて説明する  
(Describe Microsoft Purview Compliance Manager, Compliance Scores, Microsoft Priva, and the benefits that they can provide to an organization)
  - 内部の脅威から保護するためのインサイダー リスク管理ソリューションについて説明する  
(Describe insider risk management solutions to protect against internal threats)
  - 監査および eDiscovery ソリューションについて説明する  
(Describe auditing and eDiscovery solutions)
  - プライバシー管理の概念を説明する  
(Describe privacy management concepts)
  - Microsoft のプライバシー原則を説明する  
(Describe Microsoft’s privacy principles)
#### Microsoft 365 の価格とサポートを説明する (Describe Microsoft 365 pricing and support) (10–15%)
- 価格設定の計画、予測、比較を行う  
(Plan, predict, and compare pricing)
  - エンタープライズ契約、クラウド ソリューション プロバイダー (CSP)、直接請求を含む Microsoft クラウド サービスの価格モデルについて説明する  
(Describe the pricing model for Microsoft cloud services including enterprise agreements, cloud solution providers, direct billing)
  - 請求頻度や支払い方法を含む、利用可能な請求および請求管理オプションについて説明する  
(Describe available billing and bill management options including billing frequency and methods of payment)
- Microsoft 365 で利用可能なライセンス オプションを特定する  
(Identify licensing options available in Microsoft 365)
  - 利用可能な基本ライセンスと管理オプションについて説明する  
(Describe the available base licensing and management options)
  - アドオン ライセンス オプションを説明する  
(Describe add-on licensing options)
- Microsoft 365 サービス ライフサイクルを説明する  
(Describe the Microsoft 365 service lifecycle)
  - プライベート プレビュー、パブリック プレビュー、および一般提供 (GA) オプションについて説明する  
(Describe private, public preview, and general availability (GA) options)
  - ライフサイクル オプションと価格設定の相関関係を説明する  
(Describe the correlation between lifecycle options and pricing)
  - Microsoft 365 ロードマップ ポータルの使用例と機能について説明する  
(Describe use cases and capabilities for the Microsoft 365 Roadmap portal)
- Microsoft 365 サービスのサポート オファリングを説明する  
(Describe support offerings for Microsoft 365 services)
  - Microsoft 365 サービスのサポート リクエストをどのように作成するかを説明する  
(Describe how to create a support request for Microsoft 365 services)
  - 稼働時間の原則、払い戻しのためのサービス クレジット、SLA 階層、役割、および責任を含むサービス レベル アグリーメント (SLA) の概念を説明する  
(Describe service level agreements (SLAs) concepts including uptime principles, service credits for refunds, SLA tiers, roles, and responsibilities)
  - Microsoft 365 管理ポータル、もしくは Azure テナント ポータルを用いてサービスの正常性ステータスを説明する  
(Determine service health status by using the Microsoft 365 admin portal or the Azure tenant portal)
  - 組織が製品とサービスの改善について、Microsoft とやり取りする方法を説明する  
(Describe how organizations can communicate with Microsoft about product and service improvements)
