# MS-203 : Microsoft 365 Messaging
Microsoft 認定資格「Microsoft 365 Certified: Messaging Administrator Associate」の対象試験です。
- Exam MS-203: Microsoft 365 Messaging - Learn | Microsoft Docs  
https://docs.microsoft.com/learn/certifications/exams/ms-203

## 試験について (意訳)
**※2022/10/31 に英語版が改定 (アップデート) されました。**
- Study Guide - Exam MS-203: Microsoft 365 Messaging–Skills Measured  
https://query.prod.cms.rt.microsoft.com/cms/api/am/binary/RE4xqnc

この試験の受験者は、ハイブリッドおよびクラウド エンタープライズ環境でメッセージング インフラストラクチャを計画、展開、構成、トラブルシューティング、および保護するメッセージング管理者です。  
(Candidates for this exam are messaging administrators who plan, deploy, configure, troubleshoot, and secure the messaging infrastructure in hybrid and cloud enterprise environments.)

また、受信者、メールボックス、トランスポート、メール フロー、管理者の役割、脅威からの保護、コンプライアンス、移行、およびクライアント接続の管理を担当します。  
(Messaging administrators are responsible for managing recipients, mailboxes, transport, mail flow, administrative roles, threat protection, compliance, migrations, and client connectivity.)

さらに、他のワークロード管理者および Microsoft 365 エンタープライズ管理者と協力して、企業のビジネス ニーズを満たす安全なハイブリッドまたはクラウドのみのメッセージング トポロジを実装します。  
(Messaging administrators collaborate with other workload administrators and the Microsoft 365 enterprise administrator to implement a secure hybrid or cloud-only messaging topology that meets the business needs of an enterprise.)

加えて、認証の種類、ライセンス、Microsoft 365 アプリケーションとの統合、および Microsoft Entra の一部である Microsoft Azure Active Directory (Azure AD) に関する実用的な知識を持っている必要があります。  
(The messaging administrator should have a working knowledge of authentication types, licensing, integration with Microsoft 365 applications, and Microsoft Azure Active Directory (Azure AD), part of Microsoft Entra.)

### 機能グループ
#### 組織の設定とリソースの管理 (Manage organizational settings and resources) (15—20%)
- 組織の設定とクライアントの接続を計画、管理する  
(Plan and manage organizational settings and client connectivity)
  - グローバル アドレス リスト (GAL) を管理する  
  (Manage a Global Address List (GAL))
  - オフラインアドレス帳 (OAB) を管理する  
  (Manage an Offline Address Book (OAB))
  - アドレス帳ポリシーを管理する  
  (Manage address book policies)
  - 電子メール アドレス ポリシーを管理する  
  (Manage email address policies)
  - 組織信頼関係や個別の共有を計画、構成する  
  (Plan and configure organization relationships and individual sharing)
  - ハイブリッド環境の仮想ディレクトリや URL を構成する  
  (Configure virtual directories and URLs in a hybrid environment)
  - Web ポリシー上のグローバル Outlook を構成する  
  (Configure global Outlook on the Web policies)
  - Microsoft Support and Recovery Assistant (SaRA) を用いてトラブルシュートする  
  (Troubleshoot by using the Microsoft Support and Recovery Assistant (SaRA))
- 受信者のリソースを管理する  
(Manage recipient resources)
  - リソース メールボックスを管理する  
  (Manage resource mailboxes)
  - 配布リスト、動的配布リスト、メールが有効なセキュリティ グループ、Microsoft 365 グループを含むグループを管理する  
  (Manage groups including distribution lists, dynamic distribution lists, mail-enabled security groups, and Microsoft 365 groups)
  - ユーザー メールボックスを管理する  
  (Manage user mailboxes)
  - 委任を含むメールボックスのアクセス許可を管理する  
  (Manage mailbox permissions, including delegation)
  - 共有メールボックスへの変換を含む、共有メールボックスを管理する  
  (Manage shared mailboxes, including converting to a shared mailbox)
  - メール連絡先やメール ユーザーを含む連絡先を作成、管理する  
  (Create and manage contacts, including mail contacts and mail users)
  - パブリック フォルダーを作成、管理する  
  (Create and manage public folders)
#### メール トランスポート アーキテクチャの計画と管理 (Plan and manage the mail transport architecture) (30—35%)
- コネクタを計画、構成する  
(Plan and configure connectors)
  - メール フローのコネクターを計画する  
  (Plan connectors for mail flow)
  - メールをルーティングするコネクターを作成、構成する  
  (Create and configure connectors to route mail)
  - 組織間のメール フローをセキュリティで保護するためのコネクターを作成、構成する  
  (Create and configure connectors to secure mail flow between organizations)
  - Microsoft 365 Defender でコネクターの拡張フィルター処理を構成する  
  (Configure enhanced filtering for connectors in Microsoft 365 Defender)
- 内部メールとインターネット メールのフローを管理、保護する  
(Manage and secure flow of internal and internet mail)
  - メール フロー ルールを管理する  
  (Manage mail flow rules)
  - 承認済みドメインとリモート ドメインを管理する  
  (Manage accepted and remote domains)
  - ハイブリッド メール フローを管理する  
  (Manage hybrid mail flow)
  - 特定のシナリオに適した DNS MX レコードを特定する  
  (Identify the appropriate DNS MX record for a specific scenario)
  - Sender Policy Framework (SPF) および Domain-based Message Authentication、Reporting and Conformance (DMARC) を計画、実装する    
  (Plan and implement Sender Policy Framework (SPF) and Domain-based Message Authentication, Reporting and Conformance (DMARC))
  - Domain Keys Identified Mail (DKIM) を計画、実装する  
  (Plan and implement Domain Keys Identified Mail (DKIM))
- メール フローをトラブルシュートする  
(Troubleshoot mail flow)
  - アラート ポリシーを構成、管理する  
  (Configure and manage alert policies)
  - メッセージをトレースする  
  (Trace a message)
  - メッセージ トレースを分析する  
  (Analyze a message trace)
  - メッセージ ヘッダーを分析する  
  (Analyze message headers)
  - 配信不能レポート (NDR) を分析する  
  (Analyze Non-Deliverable Reports (NDRs))
  - Microsoft Remote Connectivity Analyzer を用いてトラブルシュートする  
  (Troubleshoot by using the Microsoft Remote Connectivity Analyzer)
  - ハイブリッド 環境のトランスポート ルールやキューを用いてトラブルシュートする  
  (Troubleshoot by using transport logs and queues in a hybrid environment)
#### ハイブリット構成と移行の計画と実装 (Plan and implement a hybrid configuration and migration) (15—20%)
- ハイブリッド環境を計画、実装する  
(Plan and implement a hybrid environment)
  - ハイブリッド構成ウィザード (HCW) を用いてハイブリッド展開を計画する  
  (Plan a hybrid deployment by using the Hybrid Configuration Wizard (HCW))
  - ハイブリッド環境でエッジ サーバーの役割を管理する  
  (Manage the Edge server role in a hybrid environment)
  - Hybrid Configuration Wizard を用いて環境を構成する  
  (Configure the environment by using the Hybrid Configuration Wizard)
  - Exchange Hybrid Agent をデプロイする
  (Deploy the Exchange Hybrid Agent)
  - ハイブリッド環境で Exchange 証明書のライフサイクルを管理する  
  (Manage the Exchange certificates lifecycle in a hybrid environment)
- 移行を計画、実装する  
(Plan and implement migration)
  - 最小限のハイブリッド (高速移行)、リモート移動、カットオーバー、クロステナント、ステージング、PST インポート、IMAP、Google Workspace を含む移行方法を評価する  
  (Evaluate migration methods, including minimal hybrid (express migration), remote move, cutover, cross tenant, staged, PST Imports, IMAP, and Google Workspace)
  - 移行エンドポイントを構成する  
  (Configure migration endpoints)
  - 移行バッチを作成、管理する  
  (Create and manage a migration batch)
  - テナント間の移行要件を特定する  
  (Specify cross-tenant migration requirements)
  - メールボックスの移行をトラブルシュートする  
  (Troubleshoot mailbox migration)
#### メッセージング環境のセキュア化 (Secure the messaging environment) (25—30%)
- Exchange のロールベースのアクセス許可を管理する  
(Manage role-based permissions in Exchange)
  - 注: Exchange に関連したロールのみ  
  (Note: Exchange-related roles only)
  - RBAC ロールを計画する  
  (Plan RBAC roles)
  - デフォルトおよびカスタム管理者ロール グループを管理する  
  (Manage default and custom admin role groups)
  - Manage user roles  
  (ユーザー ロールを管理する)
- Microsoft 365 Defender を用いてメッセージをセキュアにする  
(Secure messages by using Microsoft 365 Defender)
  - アンチフィッシング ポリシーを構成、管理する  
  (Configure and manage Anti-phishing policies)
  - アンチスパム ポリシーを構成、管理する  
  (Configure and manage Anti-spam policies)
  - アンチマルウェア ポリシーを構成、管理する  
  (Configure and manage Anti-malware policies)
  - セーフ アタッチメントを構成、管理する  
  (Configure and manage Safe Attachments)
  - セーフ リンクを構成、管理する  
  (Configure and manage Safe Links)
  - 脅威許可/ブロック リストを構成する  
  (Configure Tenant Allow/Block lists)
  - 検疫ポリシーを構成、管理する  
  (Configure and manage quarantine policies)
  - Microsoft 365 Defender Explorer や Advanced Hunting を用いてメッセージングの脅威を調査する  
  (Investigate messaging threats by using Microsoft 365 Defender Explorer and Advanced Hunting)
  - ユーザー報告メッセージの設定を管理する  
  (Manage user-reported message settings)
  - Configuration Analyzer を用いて設定を検証する  
  (Validate settings by using Configuration Analyzer)
- コンプライアンスを管理する  
(Manage compliance)
  - 特定のメールボックス フォルダーの保持タグやアイテム保持ポリシーを構成する  
  (Configure retention tags and retention policies for a specific mailbox folder)
  - アーカイブ メールボックスを有効化、構成する  
  (Enable and configure archive mailboxes)
  - 監査ログを分析する  
  (Analyze audit logs)
  - ジャーナル ルールを管理する  
  (Manage journal rules)
  - コンテンツ検索を管理する  
  (Manage content search)
  - コア eDiscovery ケース (高度な eDiscovery は含まない) を管理する  
  (Manage core eDiscovery cases (not including Advanced eDiscovery))
