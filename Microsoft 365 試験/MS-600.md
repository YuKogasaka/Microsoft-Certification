### 試験の対象者
この試験の対象者は、Microsoft 365 プラットフォームを使用する組織の生産性とコラボレーションのニーズに合わせて最適化されたアプリケーションとソリューションを設計、構築、テスト、および保守する Microsoft 365 開発者です。

また Microsoft ID、Microsoft Graph および Microsoft Teams アプリの構築に精通している必要があります。受験対象者は、UI 要素 (Fluent UI (Office UI Fabric) を含む)、統合ポイント (Office アドイン、SharePoint Framework、アクション可能メッセージを含む)、およびワークロード プラットフォーム ターゲットの決定に関する一般的な知識を持っている必要があります。

加えて、ソフトウェア開発のすべてのフェーズを通じて、Microsoft Teams、Office、もしくは SharePoint でソリューションを開発した経験が必要です。REST API、JSON、OAuth2、OData、OpenID Connect、Microsoft ID (Azure AD や Microsoft アカウントを含む)、Azure AD B2C、およびアクセス許可/同意の概念についての基本的な知識を持っている必要があります。

### スキルの評価
注：評価される各スキルの下に記載しているリストは、スキルの評価項目です。このリストは、決定的でも網羅的でもありません。

注：試験内容について、ほとんどの場合、プレビュー機能はカバーされず、一部の機能は GA (General Availability) となった際に試験に追加されます。

#### Microsoft ID の実装 (20-25%)
- アプリケーションを登録する
  - サポートされているアカウント タイプを決定する
  - アプリのタイプや認証フローの認証とクライアントの資格情報を選択する
  - アプリのロールを定義する
- 認証を実装する
  - エンドポイントおよびトークン キャッシュ用に Microsoft認証ライブラリ (MSAL) の Java Script 実装を構成する
  - 動的/静的アクセス権限に対するスコープを計画、構成する
  - MSAL (JavaScript) のログイン方法を利用する
- API を消費するためのアクセス権限を構成する
  - アプリに対する委任されたアクセス権限を構成する
  - アプリに対するアプリケーションのアクセス権限を構成する
  - 管理者の同意要件を特定する
- API を消費するための認証を実装する
  - 増分同意スコープを構成する
  - aquireTokenSilent/aquireToken パターンを用いて MSAL (JavaScript) を呼び出す
- API の認証を実装する
  - アクセス トークンを検証する
  - 委任されたスコープに対して効果的なアクセス権限を構成する
  - ロールを用いてアプリのアクセス権限を実装する
  - Microsoft API を呼び出すための委任されたアクセス トークンを利用する
- Microsoft Graph にアクセスするサービスを作成する
  - 証明書を用いてクライアント資格情報を構成する
  - アプリに対するアプリケーションのアクセス権限を構成する
  - アプリケーションのアクセス権限やクライアント資格情報を用いて、Microsoft Graph のアクセス トークンを取得する
  - クライアント シークレットを用いてアクセス トークンを取得する
#### Microsoft Graph を用いたアプリのビルド (15-20%)
- クエリ パラメーターにてデータの使用量を最適化する
  - $filter クエリ パラメーターを利用する
  - $select クエリ パラメーターを利用する
  - $orderby クエリ パラメーターを用いて結果をオーダーする
  - $skip や $top クエリ パラメーターを用いてページ サイズの結果を背呈する
  - $expand クエリ パラメーターを用いてリソースを展開、取得する
  - $count クエリ パラメーターを用いて、一致するリソースの総数を取得する
  - $search クエリ パラメーターを用いてリソースに対して検索する
  - 活用するための適切な Microsoft Graph SDK を決定する
- ネットワーク トラフィックを最適化する
  - 変更通知を用いて変更を監視する
  - $batch を用いて複数の要求を結合する
  - デルタ クエリを用いて変更を取得する
  - スロットリングを検出、ハンドリングする
- Microsoft Graph からユーザー データにアクセスする
  - サインインしたユーザーのプロファイルを取得する
  - 組織内のユーザー一覧を取得する
  - ユーザー プロファイルの画像を取得する
  - ユーザーのユニーク ID を基づくユーザー オブジェクトを取得する
  - ユーザー/メンバーのプロファイルを取得する
- Microsoft Graph でファイルにアクセスする
  - サインインしたユーザーの OneDrive にあるファイル一覧を取得する
  - ファイルのユニーク ID を用いてサインインしたユーザーの OneDrive からファイルをダウンロードする
  - ファイルへの相対パスを用いて、SharePoint Online サイトからファイルをダウンロードする
  - サインインしたユーザーの周りでトレンドになっているファイル一覧を取得する
  - OneDrive に大きなファイルをアップロードする
  - グループ内の所有者一覧からユーザー オブジェクトを取得し、ユーザーのファイルを取得する
- Microsoft Graph のグループ ライフサイクルを管理する
  - ID でグループに関する情報を取得する
  - グループ内のメンバー一覧を取得する
  - グループの所有者の一覧を取得する
  - サインインしたユーザーがメンバーであるグループ一覧を取得する
  - サインインしたユーザーが所有者であるグループ一覧を取得する
  - グループをプロビジョニングする
  - グループを持つチームをプロビジョニングする
  - グループを削除する
#### SharePoint の拡張とカスタマイズ (15-20%)
- SharePoint Framework (SPFx) Web パーツのコンポーネントを説明する
  - SPFx Web パーツ プロジェクトを作成するための適切なツールを特定する
  - クライアント サイド Web パーツのプロパティを説明する
  - クライアント サイド Web パーツで FluentUI (Office UI Fabric) を説明する
  - アプリ ページを利用する場合を説明する
  - アプリ ページと Web パーツを区別する
  - レンディング フレームワーク オプションを説明する
  - SharePoint Online のブランディングとテーマを説明する
- SPFx 拡張機能を説明する
  - SPFx 拡張機能プロジェクトを作成するために適切なツールを特定する
  - Application Customizer からページ プレースホルダーを説明する
  - ListView Command Set 拡張機能を説明する
  - Field Customizer 拡張機能を説明する
- SPFx ソリューションをパッケージ、デプロイするための処理を説明する
  - デプロイメント用のパッケージを準備するためのオプションを説明する
  - ソリューションをパッケージングするオプションを説明する
  - テナント スコープのソリューション デプロイメントの要件を説明する
  - 分離された Web パーツの要件を説明する
  - ソリューションをデプロイするためのオプションを説明する
- Microsoft Graph の消費について説明する
  - MSGraphClient オブジェクトの目的を説明する
  - Microsoft Graph にアクセス権限を付与する方法を説明する
- SPFx 内から Azure AD で保護されたサードパーティ API の消費について説明する
  - AadHttpClient オブジェクトの目的を説明する
  - サードパーティ API を利用するための権限を付与する方法を説明する
- Teams Tab として Web パーツを説明する
  - SPFx を用いてどのように Microsoft Teams Tab をビルドするかを説明する
  - Teams Tab になる SPFx Web パーツを作成するための考慮事項を説明する
  - Teams Tab として SPFx Web パーツのデプロイ オプションを説明する
#### Teams の拡張 (20-25%)
- Microsoft Teams アプリ マニフェストを作成する
  - App Studio を用いて アプリのマニフェストを構成する
  - Teams に SPFx Web パーツをデプロイするためにアプリの マニフェストを手動で作成する
  - Microsoft Teams アプリのアプリ パッケージを作成する
- Teams アプリをデプロイする
  - Teams アプリのデプロイ オプションを説明する
  - Microsoft Teams でアプリをサイドロードする
  - 組織のアプリ カタログに Teams アプリを公開する
- タスク モジュールを作成、利用する
  - カード ベースのタスク モジュールを作成する
  - iframe ベースのタスク モジュールを作成する
  - タブからタスク モジュールを呼び出す
  - Bot からタスク モジュールを呼び出す
  - タスク モジュールの呼び出しをチェーンする
- Webhook を作成する
  - 送信 webhook を作成する
  - 着信 Webhook を作成する
- カスタム Teams タブを実装する
  - 個人用タブを作成する
  - チャネル/グループ タブを作成する
  - ディープ リンクにタブを作成する
  - タブに対してシングル サインオン認証を実装する
- メッセージ拡張機能を作成する
  - メッセージ拡張機能の検索コマンドを作成する
  - アダプティブ カードを用いてメッセージ拡張機能のアクション コマンドを作成する
  - パラメーターを用いてメッセージ拡張機能のアクション コマンドを作成する
- 会話型 Bot を作成する
  - 個人用 Bot を作成する
  - グループ/チャネル Bot を作成する
  - Bot のプロアクティブ メッセージを利用する
  - Bot からアクション可能メッセージを送信する
  - Bot に対してシングル サインオン認証を実装する
- Microsoft Graph を利用して Teams を操作する
  - チームを作成する
  - チームを一覧表示する
  - ビルトイン タブを構成する
  - ユーザーにアクティビティ フィード通知を送信する
  - リソースを有効化する
#### Office の拡張 (15-20%)
- 基本的なコンポーネントや Office アドインのタイプを説明する
  - タスク ペインやコンテンツ アドインを説明する
  - ダイアログ ボックスを説明する
  - カスタム関数を説明する
  - アドイン コマンドを説明する
  - Office アドイン マニフェストの目的を説明する
- Office JS API を説明する
  - Office アドイン プログラミング モデルを説明する
  - Office アドイン開発ツールを説明する 
  - Excel JavaScript API の機能を説明する
  - Outlook JavaScript API の機能を説明する
  - Word JavaScript API の機能を説明する
  - PowerPoint JavaScript API の機能を説明する
  - カスタム関数の機能を説明する
- アドインの開発を説明する
  - 永続化状態と設定のオプションを説明する
  - Office アドインの Fluent UI (Office UI Fabric) を説明する
  - Office アドインで Microsoft Graph を利用する場合を説明する
  - Office アドインで Microsoft Graph を利用する場合の承認について説明する
- テスト、デバッグ、デプロイメント オプションを説明する
  - select deployment options based on requirements
  - Office アドインのテストとデバッグのコンセプトを説明する
- アクション可能メッセージを説明する
  - アダプティブ カードを持つアクション可能メッセージ機能を説明する
  - アクション可能メッセージを更新するためのシナリオを説明する
