### 試験の対象者
この試験の対象者は、SQL Server と Azure Data Services で構築されたオンプレミスおよびクラウドのリレーショナル データベースを管理するデータベース管理者とデータ管理スペシャリストです。

Azure Database Administrator は、Azure DataServices と SQL Server 上に構築されたクラウド ネイティブおよびハイブリッド データ プラットフォーム ソリューションの運用面を実装および管理します。また、さまざまな方法とツールを用いた運用管理目的での T-SQL の使用に関する知識の適用など、日常の操作を実行します。

この役割は、最新のリレーショナル データベース ソリューションの管理、可用性、セキュリティ、パフォーマンスの監視と最適化を担当します。また、Azure Data Engineer の役割と連携して、データ プラットフォーム ソリューションの運用面を管理します。

対象者は、DP-900 : この役割の候補者は、DP-900でカバーされているすべての概念を理解する必要があります。

### スキルの評価
注：評価される各スキルの下に記載しているリストは、スキルの評価項目です。このリストは、決定的でも網羅的でもありません。

注：試験内容について、ほとんどの場合、プレビュー機能はカバーされず、一部の機能は GA (General Availability) となった際に試験に追加されます。

#### データ プラットフォーム リソースの計画と実装 (15-20%)
- 手動の方法を用いてリソースをデプロイする
  - 選択したプラットフォームにデータベース オファリングをデプロイする
  - カスタマイズ化したデプロイメント テンプレートを構成する
  - ハイブリッドおよび IaaS デプロイメント用のパッチやアップデートを適用する
- 特定の要件に基づいた適切なデータベース オファリングを推奨する
  - デプロイメント要件を評価する
  - 利用可能なデータベース オファリングの機能上の利点/影響を評価する
  - 利用可能なデータベース オファリングのスケーラビリティを評価する
  - 利用可能なデータベース オファリングの HA/DR を評価する
  - 利用可能なデータベース オファリングのセキュリティ面を評価する
- スケールやパフォーマンス リソースを構成する
  - スケールやパフォーマンスに対する Azure SQL データベースを構成する
  - スケールやパフォーマンスについて Azure SQL Managed Instances を構成する
  - スケールやパフォーマンスに対する Azure VM 上の SQL Server に構成する
  - リソース要件を計算する
  - データベース シャーティングのようなデータベース パーティショニング技術を評価する
  - SQL Data Sync をセットアップする
- Azure への移行戦略を評価する
  - 移行要件を評価する
  - オフライン、もしくはオンラインの移行戦略を評価する
  - アップグレード要件を評価する
  - オフライン、もしくはオンラインのアップグレード戦略を評価する
- Azure に移行するための移行、もしくはAzure に移行するためのアップグレード戦略を実装する
  - オンライン マイグレーション戦略を実装する
  - オフライン移行戦略を実装する
  - オンライン アップデート 戦略を実装する
  - オフライン アップデート 戦略を実装する
#### セキュアな環境の実装 (15-20%)
- プラットフォームおよびデータベースツールを用いてデータベース認証を構成する
  - Azure AD 認証を構成する
  - Azure AD ID からユーザーを作成する
  - セキュリティ プリンシパルを構成する
- プラットフォームおよびデータベース ツールを用いてデータベースの承認を構成する
  - グラフィカル ツールを用いてデータベースおよびオブジェクト レベルのアクセス権限を構成する
  - すべての保護対象に対して最小特権の原則を適用する
- 保存データのセキュリティを実装する
  - 透過的なデータ暗号化を実装する (TDE)
  - オブジェクト レベル暗号化を実装する
  - 動的データ マスクを実装する
  - Azure VM に Azure Key Vault とディスク暗号化を実装する
- 転送中のデータのセキュリティを実装する
  - サーバーおよびデータベース レベルのファイアウォール ルールを構成する
  - Always Encrypted を実装する
- 機密データのコンプライアンス管理を実装する
  - データ分類戦略を適用する
  - サーバーとデータベースの監査を構成する
  - データ変更追跡を実装する
  - 脆弱性評価を実行する
#### 運用リソースの監視と最適化 (15-20%)
- アクティビティやパフォーマンスを監視する
  - 運用パフォーマンスのベースラインを準備する
  - パフォーマンス メトリックのソースを決定する
  - パフォーマンス メトリックを解釈する
  - Intelligent Insights for Azure SQL データベースとマネージド インスタンスを用いてデータベースのパフォーマンスを評価する
  - インフラストラクチャ、サーバー、サービス、およびデータベース レベルでアクティビティとパフォーマンスを構成、監視する
- パフォーマンス関連のメンテナンス タスクを実装する
  - インデックス メンテナンス タスクを実装する
  - 統計メンテナンス タスクを実装する
  - データベースの自動チューニングを構成する
  - データベース管理タスクを自動化する
  - ストレージ容量を管理する
- パフォーマンス関連の問題を特定する
  - パフォーマンス データを収集する Query Store を構成する
  - ブロッキングの原因となるセッションを特定する
  - データベースとログの増大/フラグメントを評価する
  - パフォーマンス関連のデータベース構成パラメーターを評価する
- 最適なパフォーマンスのためにリソースを構成する
  - ストレージやインフラストラクチャ リソースを構成する
  - パフォーマンスのためにサーバーとサービス アカウントの設定を構成する
  - パフォーマンス用に Resource Governor を構成する
- 最適なパフォーマンスを得るためにユーザー データベースを構成する
 - データベース スコープの構成を実装する
 - スケーリング関するコンピューティング リソースを構成する
 - Intelligent Query Processing (IQP) を構成する
#### クエリ パフォーマンスの最適化 (5-10%)
- クエリ プランをレビューする
  - 実行プランの適切なタイプを決定する
  -  実行計画の問題領域を特定する
  - Query Store からクエリ プランを抽出する
- パフォーマンスの改善を評価する
  - クエリのパフォーマンス情報を収集するための適切な動的管理ビュー (DMV) を決定する
  - DMV を用いたパフォーマンスの問題を特定する
  - クエリのインデックス変更を特定、実装する
  - リソースの使用状況に基づいたクエリ構成の変更を推奨する
  - クエリ パフォーマンスのヒントの使用を評価する
- データベース テーブルとインデックス設計をレビューする
  - データの重複によるデータ品質の問題を特定する
  - データベース テーブルの通常の形式を特定する
  - パフォーマンスのインデックス設計を評価する
  - 列に定義されたデータ型を検証する
  - ファイル グループを含むテーブルとインデックスのストレージを推奨する
  - テーブルのパーティショニング戦略を評価する
  - テーブルとインデックスの圧縮の使用を評価する
#### タスク自動化の実行 (10-15%)
- スケジュールされたタスクを作成する
  - 定期的なメンテナンス ジョブのスケジュールを管理する
  - マルチサーバー自動化を構成する
  - タスクの成功/失敗/未完了の通知を構成する
- アラートと通知戦略を評価、実装する
  - メトリックに基づいたイベント通知を作成する
  - リソースのイベント通知を作成する
  - サーバー構成変更アラートを作成する
  - イベント通知に応答するタスクを作成する
- Azure のタスクを管理、自動化する
  - リソースの自動デプロイメント手法を実行する
  - バックアップを自動化する
  - パフォーマンス チューニングとパッチ適用を自動化する
  - 自動評価モードを用いてポリシーを実装する
#### 高可用性と災害復旧 (HADR) 環境の計画と実装 (15-20%)
- データ プラットフォーム ソリューション用の HADR 戦略を推奨する
  - RPO/RTO 要件に基づいた HADR 戦略を推奨する
  - ハイブリッド デプロイメント用の HADR を評価する
  - Azure 特有の HADR ソリューションを評価する
  - HADR ソリューションのリソースを特定する
- プラットフォーム、OS、データベース ツールを用いて HADR 戦略をテストする
  - フェールオーバーを用いて HA をテストする
  - フェールオーバーかリストアを用いて DR をテストする
- データベース ツールを用いてデータベースをバックアップやリストアを実行する
  - オプションを用いてデータベースのバックアップを実行する
  - オプションを用いてデータベースのリストアを実行する
  - ポイント イン タイムにデータベースのリストアを実行する
  - 長期的なバックアップ保有期間を構成する
- OS、プラットフォームやデータベース ツールを用いて HA/DR を構成する
  - レプリケーションを構成する
  - Alwways On Availability Group を作成する
  - 自動フェールオーバー グループを構成する
  - Alwways On Availability Group にデータベースを統合する
  - Windows Server Failover Cluster のクォーラム オプションを構成する
  - Alwways On Availability Group リスナーを構成する
  - Azure VM 上にフェールオーバー クラスター インスタンスを構成する
#### T-SQL を用いた管理の実行 (10-15%)
- システムの正常性を調査する
  - DMV を用いてデータベースの正常性を評価する
  - DMV を用いてサーバーの正常性を評価する
  - DBCC を用いてデータベース一貫性チェックを実行する
- T-SQL を用いてデータベース構成を監視する
  - 適切なデータベースの自動各聴講生を評価する
  - データベースの空き容量をレポートする
  - データベース構成オプションをレビューする
- T-SQL を用いてデータベースのバックアップ、リストアを実行する
  - Always On Availability Group 用のデータベースを準備する
  - トランザクション ログ バックアップを実行する
  - ユーザー データベースのリストアを十個する
  - オプションを用いてデータベース バックアップを実行する
- T-SQL を用いて認証を管理する
  - 証明書を管理する
  - セキュリティ プリンシパルを管理する
- T-SQL を用いて承認を管理する
  - データベース オブジェクトにアクセスするたえにユーザーのアクセス権限を構成する
  - カスタム ロールを用いてアクセス権限を構成する
